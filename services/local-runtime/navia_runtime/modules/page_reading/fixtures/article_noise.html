Companion Reading Architecture

Navia is a browser companion reading runtime that extracts page facts before the agent writes an answer.

The A module is the perception layer. It cleans visible text, preserves paragraph identifiers, and marks source references for every downstream digest item.

According to the V1.2 plan, high-signal output must include source coverage, grounding completeness, jumpback coverage, and a deterministic quality report.

Implementation Steps

First, the page is converted into structured paragraphs. Second, noisy navigation and recommendation blocks are filtered. Third, grounded facts are distilled for D and C modules.

Evidence matters because every key fact must be traceable back to textQuote and fallbackText when DOM selector jumpback fails.
