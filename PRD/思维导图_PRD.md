# 思维导图 PRD

## 1. 背景
Navia 需要提供一种轻量的结构化展示能力，用于将当前上下文快速整理为右向思维导图，帮助用户更快理解内容脉络、提炼重点并回看结果。

本功能的定位不是编辑器，而是一个“点击即生成”的结果型视图：
- 用户点击思维导图选项后，系统自动生成导图
- 前端直接渲染右向思维导图结果
- 导图仅用于展示，不支持编辑、拖拽或折叠

## 2. 目标
- 用户点击思维导图选项后，可以立即看到结构化导图结果。
- 系统优先使用 LLM 生成导图结构，提高结果质量。
- 当 LLM 不可用或输出异常时，系统仍能返回可展示的 fallback 结果。
- 导图在 Side Panel 场景下保持轻量、清晰、加载快。

## 3. 产品定位
### 3.1 核心定位
- 结果型视图
- 只读展示
- 轻量 SVG 渲染
- LLM 优先，fallback 保底

### 3.2 非目标
- 不做节点编辑
- 不做拖拽重排
- 不做折叠/展开
- 不做导出图片
- 不做多种导图样式切换
- 不做手动输入内容作为主流程

## 4. 使用场景
### 4.1 点击即生成
用户在 Chatbot 或相关入口中点击“思维导图”选项后，系统根据当前上下文自动生成对应导图并展示结果。

### 4.2 结果回看
用户生成导图后，可以在当前会话中再次查看该导图结果。

### 4.3 内容整理
系统基于当前上下文自动提炼结构化信息，并转化为可视化导图，帮助用户快速理解内容结构。

## 5. 功能范围
### 5.1 触发方式
- 用户点击“思维导图”选项后，系统立即发起生成流程。
- 本功能不要求用户先手动粘贴内容。
- 本功能不暴露独立内容输入步骤作为主流程。

### 5.2 生成来源
- 导图内容由系统根据当前上下文、会话内容或上游输入自动确定。
- 输入来源属于上游流程能力，不作为思维导图页面的独立交互项。

### 5.3 展示
- 生成结果直接渲染为右向思维导图。
- 导图只读，不支持编辑。
- 导图支持在当前页面内滚动查看。

### 5.4 恢复与持久化
- 生成结果保存为 artifact。
- 会话恢复时可回看历史导图结果。
- artifact 与现有会话系统兼容。

## 6. 用户流程
### 6.1 基本流程
1. 用户点击“思维导图”选项
2. 系统根据当前上下文发起生成
3. Runtime 返回结构化导图结果
4. 前端直接渲染右向思维导图
5. 导图结果保存为 artifact

### 6.2 失败流程
1. 用户点击“思维导图”选项
2. LLM 不可用、返回异常或 JSON 解析失败
3. 系统自动切换到本地 fallback
4. 页面仍展示可用导图结果

## 7. 功能需求
### 7.1 生成入口
- 提供“思维导图”入口选项。
- 用户点击后立即触发生成。
- 不要求用户先输入文本或粘贴内容。

### 7.2 导图结构
导图 JSON 必须满足以下结构：

```json
{
  "version": "1.0",
  "title": "导图标题",
  "root": {
    "id": "node_1",
    "label": "根节点",
    "children": []
  }
}
```

节点字段：
- `id`
- `label`
- `note`，可选
- `children`，可选

### 7.3 生成策略
- 优先调用 LLM 生成树状金字塔 JSON。
- 只接受 JSON 输出。
- 支持从 fenced code block 中提取 JSON。
- 支持从混合文本中提取 JSON。
- 若无法解析，转 fallback。

### 7.4 fallback 策略
- 无 API Key 时走 fallback。
- model 或 base_url 缺失时走 fallback。
- LLM 请求异常时走 fallback。
- JSON 解析失败时走 fallback。

### 7.5 展示策略
- 采用 SVG 右向树状布局。
- 根节点在左侧。
- 子节点按深度向右展开。
- 节点之间使用曲线连接线。
- 节点使用圆角矩形。
- 节点文本使用 SVG `text` / `tspan` 渲染。
- 不使用 HTML 注入方式渲染节点文本。

## 8. 规则与约束
### 8.1 结构约束
- 最多 4 层。
- 最多 40 个节点。
- 单个 label 最多 48 个字符。

### 8.2 文本约束
- label 过长时截断。
- 空 label 使用“未命名”。
- title 缺失时使用 fallback 标题。

### 8.3 视觉约束
- 适配 Chrome Side Panel 宽度。
- 支持横向滚动。
- 避免节点重叠。
- 避免文本溢出造成布局破坏。

### 8.4 数据约束
- 生成结果必须可序列化保存为 artifact。
- artifact 中应标记导图格式。
- artifact 与会话系统兼容。

## 9. 接口需求
### 9.1 生成接口
`POST /v1/mindmap/generate`

请求字段：
- `session_id`
- `content`
- `title`，可选
- `llm_config`，可选
- `request_id`，可选

返回字段：
- `mindmap`
- `artifact`
- `generation`

### 9.2 artifact 约定
- `type = pyramid_mindmap`
- `metadata.format = pyramid-json`
- `metadata.generation` 记录生成方式
- `metadata.nodesCount` 记录节点数量
- `metadata.version` 记录导图版本

## 10. 异常与兜底
### 10.1 空内容
- `content` 为空时返回请求无效。
- 明确提示缺少内容。

### 10.2 LLM 异常
- 不中断整体流程。
- 自动使用 fallback。

### 10.3 JSON 异常
- 优先从 code block 或正文中提取。
- 仍失败则 fallback。

### 10.4 前端展示异常
- 导图解析失败时显示错误提示。
- 不影响当前页面其他功能。

## 11. 验收标准
- 用户点击“思维导图”选项后能看到导图结果。
- LLM 正常时可输出结构化 JSON 导图。
- LLM 失败时仍能看到 fallback 导图。
- 导图可在页面中正确渲染为右向树状 SVG。
- 导图结果可作为 artifact 持久化。
- 空内容不可生成。
- 节点层级、节点数、label 长度受到限制。
- 页面不因长文本或复杂结构而崩坏。

## 12. 产品价值
- 把“内容理解”从纯文本输出升级为结构化视觉输出。
- 降低长内容的理解成本。
- 提供一种轻量但稳定的知识整理方式。
- 让导图能力成为 Navia 的可复用 artifact 类型。

## 13. 后续扩展方向
- 节点折叠 / 展开
- 导出图片
- 更丰富的样式主题
- 支持更多输入源
- 生成后局部重写
- 历史导图列表
- 更强的 prompt 模板化

## 14. 版本说明
- 本 PRD 描述的是当前思维导图能力的 v1 方案。
- 当前版本重点是“可用、稳定、可展示”。
- 暂不纳入编辑型、协作型和导出型能力。
