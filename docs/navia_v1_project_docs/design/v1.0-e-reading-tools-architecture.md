# V1.0-E 网页伴读工具目标架构

版本：V1.0-E Architecture Draft  
日期：2026-06-01  
适用阶段：V1.0-E 网页伴读工具

---

## 1. 阶段目标

V1.0-E 的目标是把 V1.0-D 已完成的 Chrome Side Panel 与 PageContext 链路升级为可用的网页伴读体验：

```text
Chrome 当前网页
-> Content Script 抽取 PageContext
-> Side Panel Chatbox
-> POST /v1/chat/stream
-> AgentCore TurnRunner
-> IntentRouter
-> Governance hooks
-> ToolExecutor
-> summarize / answer / explain selection / mindmap
-> SSE response
-> Side Panel 展示答案、摘要或 Mermaid mindmap
-> Trace 可按 turn_id 审计
```

V1.0-E 是 V1 用户可感知体验的核心阶段。完成后，用户应该能在 Chrome 插件中基于当前网页完成基础文字对话、网页摘要和 Mermaid 思维导图生成。

---

## 2. 范围边界

### 2.1 必须实现

- Side Panel 真实 Chatbox，不再只是 shell。
- Chatbox 消费 `/v1/chat/stream` SSE。
- Runtime 工具必须使用当前 session 的 `activePage`。
- 至少支持以下网页伴读意图：
  - 总结当前网页。
  - 基于当前网页回答问题。
  - 解释当前选中文本。
  - 生成 Mermaid mindmap。
- 每个 turn 产生 `session_id`、`turn_id`、`tool_call_id`、`artifact_id`、AgentEvent 和 trace。
- 每个工具执行必须经过 PreToolUse / PostToolUse governance hooks。
- Mermaid 必须可验证；失败时最多 repair 1 次，仍失败则返回 `MERMAID_VALIDATION_FAILED`。
- Side Panel 展示 Runtime offline、PageContext missing、streaming、tool running、error 等状态。
- Missing activePage 必须返回既有错误码 `PAGE_CONTEXT_REQUIRED`，不得生成假摘要、假回答或假 artifact。

### 2.2 明确不实现

- 不接 RAG / 长期记忆。
- 不接 MCP / Skill。
- 不做多 Agent。
- 不做浏览器自动点击、自动导航或自动操作。
- 不联网搜索。
- 不默认读取本地文件。
- 不做 YouTube 站内深度嵌入。
- 不做网页贴边悬浮球。
- 不做真实本地模型微调。

## 3. 目标架构

### 3.1 前端层

```text
apps/chrome-extension
├── entrypoints/background
├── entrypoints/content
├── entrypoints/sidepanel
└── src/pageContext.ts
```

V1.0-E 前端新增职责：

- Side Panel 维护 UI 状态，不维护 AgentCore 状态。
- 当前 session id 可缓存在 `chrome.storage`，但最终状态以 Runtime 为准。
- 读取当前网页后提交 PageContext。
- 发送用户消息到 `/v1/chat/stream`。
- 解析 SSE event：
  - `state.transition`
  - `intent.detected`
  - `budget.checked`
  - `tool.started`
  - `tool.done`
  - `artifact.created`
  - `response.delta`
  - `response.done`
  - `error`
- 未识别 SSE event 时必须忽略或记录为 debug 状态，不得让 UI 崩溃。
- 将 assistant response 和 artifact 展示在 Chatbox。

前端不可做：

- 不在 extension background 中执行 Agent loop。
- 不绕过 Runtime 直接生成摘要或 mindmap。
- 不在前端保存完整长期知识库。

### 3.2 Runtime 层

```text
services/local-runtime/navia_runtime
├── app.py
├── agent.py
├── stores.py
├── intent.py
├── tools.py
├── governance.py
├── state_machine.py
└── contracts.py
```

V1.0-E Runtime 新增职责：

- `SessionStore` 保存 `activePage` 的完整 PageContext 最小版本。
- `TurnRunner` 在 turn 开始时读取当前 `activePage`。
- `IntentRouter` 识别摘要、问答、选区解释、mindmap。
- `ToolRegistry` 注册网页伴读工具。
- `ToolExecutor` 创建 ToolCallRecord，并通过 governance hooks。
- `EventStore` 记录工具、artifact、错误和状态迁移。
- `chat/stream` 统一作为网页伴读入口。

### 3.3 工具层

V1.0-E 工具必须返回统一 `ToolResult` envelope，不得直接返回自由文本。

#### summarize_page

输入：

- `session_id`
- `turn_id`
- `page_id`
- `title`
- `url`
- `cleaned_text`
- `headings`

输出：

- 简短总览。
- 关键要点。
- 页面结构。
- 适合继续追问的问题。
- `ArtifactRecord(type="summary")`。

V1.0-E 可先使用 deterministic / rule-based 摘要，不依赖真实 LLM。

#### answer_from_page

输入：

- 用户问题。
- 当前 PageContext。
- 简单 chunk 列表或段落片段。

输出：

- 只基于当前网页的回答。
- 引用来源 chunk 或 heading path。
- 无法从网页判断时明确说明。
- `ArtifactRecord(type="answer")`。

V1.0-E 不做语义检索；可使用关键词和标题/段落匹配作为 deterministic baseline。

#### explain_selection

输入：

- `selected_text`
- surrounding context optional。

输出：

- 对选中文本的解释。
- 如果没有选区，返回 recoverable error 或引导用户先选择文本。

#### generate_mindmap

输入：

- `headings`
- `cleaned_text`
- optional summary。

输出：

- Mermaid mindmap 或 flowchart 文本。
- `ArtifactRecord(type="mindmap", metadata.format="mermaid")`。
- validator 结果。

建议 V1.0-E 采用 deterministic fallback：

```text
headings -> bounded tree -> Mermaid mindmap
```

约束：

- 最大节点数：40。
- 最大深度：4。
- Mermaid repair 次数：最多 1 次。
- 失败返回 `MERMAID_VALIDATION_FAILED`，不得无限重试。

Mermaid validation / repair 详情不得新增 ad-hoc event string。V1.0-E 采用以下承载方式：

- validation / repair 结果写入 `tool.done.data.validation` 或 `tool.done.data.repair`。
- 最终 Mermaid 信息写入 `artifact.created.data.metadata` 和 `ArtifactRecord.metadata`。
- 失败使用既有 `error` event，错误码为 `MERMAID_VALIDATION_FAILED`。

## 4. 数据流

### 4.1 PageContext 提交流

```text
Side Panel
-> chrome.tabs.sendMessage(content script)
-> extractPageContext(document)
-> POST /v1/page/context
-> Runtime normalize/hash/store activePage
-> EventStore page.context.received
```

### 4.2 Chat Turn 流

```text
User message
-> POST /v1/chat/stream
-> create turn_id / trace_id / request_id
-> StateMachine transition
-> IntentRouter
-> Budget check
-> ToolExecutor
-> ToolResult
-> ArtifactRecord
-> response.delta / response.done SSE
-> EventStore trace
```

### 4.3 Trace 审计流

```text
GET /v1/sessions/{session_id}/trace?turn_id=...
-> EventStore events
-> state.transition
-> intent.detected
-> budget.checked
-> tool.started / tool.done / tool.denied
-> artifact.created
-> response.done
```

## 5. API 影响

### 5.1 继续使用现有入口

V1.0-E 不新增独立 direct API 作为主链路，优先统一走：

```text
POST /v1/chat/stream
```

摘要、问答、解释、mindmap 均作为工具能力暴露给 AgentCore。

### 5.2 暂不新增

以下 API 暂不在 E 阶段主线实现：

```text
POST /v1/page/summarize
POST /v1/page/mindmap
```

如后续 H 阶段需要 direct artifact API，可在合同审计后补。

### 5.3 需要补充的事件

V1.0-E 不新增 AgentEvent 类型，避免产生 ad-hoc event string。

V1.0-E 复用既有事件：

- `artifact.created`
- `tool.done`
- `error`

事件承载规则：

- Mermaid validation / repair 详情放在 `tool.done.data`、`artifact.created.data.metadata` 或 `ArtifactRecord.metadata`。
- Missing activePage 使用 `error` event，`error.code = "PAGE_CONTEXT_REQUIRED"`。
- Tool failure 使用 `tool.done` with failed status，并可同时产生 `error` event。

## 6. 前端体验目标

Side Panel 必须从 shell 升级为可用 Chatbox：

- 顶部展示 Runtime 状态与当前页面状态。
- 页面上下文未提交时，聊天输入不可直接进入“基于网页回答”的假成功。
- 支持快捷动作：
  - 总结。
  - 问答。
  - Mindmap。
  - 解释选区。
- 支持手动输入问题。
- 展示 streaming response。
- 展示工具运行状态。
- 展示 Mermaid artifact。

Mermaid 展示策略需要在开发前确认：

- 已选策略：前端引入 `mermaid` 包进行真实渲染。
- Runtime 生成并验证 Mermaid source。
- Frontend 渲染 Mermaid；渲染失败时展示 source fallback 和错误状态。
- 如果后续实现未完成真实渲染，则 V1.0-E 只能声明 Mermaid source artifact，不得声明 Mermaid visual acceptance 或 V1 complete。

## 7. 真实数据验收架构

V1.0-E 必须使用真实数据验收，不允许只用 mock：

- 使用 V1.0-0 已沉淀的真实来源 HTML fixtures：
  - `article.html`
  - `docs.html`
  - `github_readme.html`
- 使用至少一个真实 Chrome tab。
- 使用真实 Side Panel。
- 使用真实 `/v1/chat/stream` SSE。
- 使用真实 Runtime EventStore trace。
- 使用真实 Mermaid validation。

验收主路径：

```text
Load unpacked extension
-> 打开真实网页
-> Side Panel online
-> 读取并提交当前页面
-> 点击“总结”
-> 收到摘要
-> 提问一个页面相关问题
-> 收到基于页面的回答
-> 点击“Mindmap”
-> 看到 Mermaid artifact
-> trace 中可按 turn_id 还原工具和 artifact
```

## 8. 关键风险

| 风险 | 等级 | 处理 |
|---|---|---|
| Mermaid 前端真实渲染需要新增依赖 | 重大 | 已选择前端 `mermaid` 依赖策略；实现阶段安装并接入，失败时 source fallback |
| 当前工具仍是 mock，未消费真实 PageContext | 重大 | E 阶段首要任务是让工具从 activePage 读取真实数据 |
| Side Panel 尚未消费 SSE | 重大 | E 阶段必须实现 SSE parser 和 streaming UI |
| ArtifactRecord 还不完整 | 重大 | E 阶段必须补 artifact 创建、事件和 trace |
| deterministic 问答质量有限 | 一般 | V1.0-E 先保证可追踪和不胡编，后续再接模型 |
| 长页面摘要可能质量差 | 一般 | E 阶段先做长度预算和 heading/paragraph baseline |

## 9. 架构结论

V1.0-E 应继续坚持 contract-first：

```text
不新增独立业务捷径。
不让前端承担 AgentCore。
不绕过 governance。
不把 mock response 当作网页伴读。
所有网页伴读结果必须能从 PageContext、ToolResult、ArtifactRecord 和 EventStore trace 追溯。
```

V1.0-E 合同闭环结论：

```text
Mermaid: Runtime validates source, frontend renders with mermaid dependency, render failure shows source fallback.
Error: missing activePage uses PAGE_CONTEXT_REQUIRED.
Events: no new AgentEvent types; use artifact.created / tool.done / error.
Artifact: no top-level format; use metadata.format.
Selection explain: ArtifactRecord(type="answer", source="selection").
```
