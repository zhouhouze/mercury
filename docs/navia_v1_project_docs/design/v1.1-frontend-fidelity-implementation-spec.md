## 1. 目标与约束

本文是 V1.1 的实现级规格，补充 `v1.1-frontend-fidelity-architecture.md` 和 `stage-gates/v1.1-frontend-fidelity.md`。它的目标是让后续工程实现不需要再决定组件边界、token 口径、截图验收范围和文件级改动顺序。

V1.1 只做前端体验高保真：

- 保留页面内悬浮球、hover 小长条、网页内 AI 双轨面板、push / overlay / resize / collapse recovery。
- 保留 PageContext、Runtime health、SSE Chat、Mermaid iframe、Session restore。
- 不修改 Runtime API、AgentEvent、ToolResult、PageContext 和 Session 数据合同。
- 不新增 MCP、Skill、RAG、多 Agent、浏览器自动操作。
- Chrome Side Panel 只作为调试入口，不参与 V1.1 高保真通过声明。

默认产品决策：

- ToolDock 在 V1.1 只做视觉占位；未实现入口必须 disabled 或 placeholder，不触发 Runtime 工具。
- Overlay 背景默认不可交互，除非 Figma 或人类确认允许底层网页点击。
- `⌘M` 在 V1.1 只作为 hover strip 提示；快捷键实现不是必需项。
- `440px / 50vw / >52vw / <48vw / max 80vw / <900px overlay` 继续沿用 PRD 口径。
- Mindmap / Mermaid artifact 视觉实现后续专项处理，不阻塞 V1.1-B/C 的通用 UI fidelity 工作。

当前 Figma Make 通过 MCP 读取不稳定。V1.1 采用双路径冻结视觉基线：

- 首选：普通 Figma `/design` 节点可被 Figma MCP 读取。
- 备选：使用 Chrome CLI 对 Figma Make live preview 做截图硬切，并在 `design/v1.1-figma-baseline/reviewed/` 完成人工复核。

本文中的视觉 token 只是默认实现草案；一旦获得已复核截图或普通 Figma `/design` 节点，必须以视觉基线覆盖默认 token，并在 stage-gate 中记录差异。

## 5. Component Specs

### 5.1 FloatingEntry

职责：

- 显示贴边浮动球。
- 支持上下拖动。
- 根据指针位置选择左/右吸附边。
- Hover 时显示小长条。
- 展开状态下隐藏 hover 小长条。

状态：

| 状态 | DOM / dataset | 视觉 |
|---|---|---|
| 默认 | `data-open=false` | 44px 圆形按钮，品牌色，贴边 |
| hover | `:hover` | 外圈高亮，小长条伸出，显示 `⌘M` |
| drag | pointer capture | y 值跟随指针，释放后吸附最近边 |
| open | `data-open=true` | 小长条隐藏，面板可见 |

验收：

- 默认态不遮挡网页主内容。
- Hover 小长条方向随 `data-side` 改变。
- 小长条是主要打开热区。
- 拖动后 `localStorage` 记住 side/y。

### 5.2 PanelShell

职责：

- 控制面板固定定位、左右展开、push / overlay 和 resize。
- 在 push 模式修改 `document.documentElement` margin。
- 在 overlay 模式清除 push margin。
- 收起时恢复网页原始 layout style。

状态：

| 状态 | 触发 | 规则 |
|---|---|---|
| closed | 默认或 collapse | 面板移出视口，恢复网页布局 |
| push narrow | width = 440px | 网页向相反方向挤压 |
| push half | width <= 52vw | 网页继续挤压 |
| overlay | width > 52vw 或 viewport < 900px | 网页不继续挤压，面板覆盖 |
| return push | overlay 拖回 < 48vw | 恢复挤压 |

验收：

- `data-mode` 与 `panelLayout.resolveLayoutMode()` 一致。
- overlay 模式不得保留 html margin。
- 收起后 html margin / transition 恢复原值。

### 5.3 LeftRail

职责：

- 呈现 Navia avatar / collapse 入口。
- 显示最小 runtime 状态 cue。
- 与 ChatWorkspace 保持固定边界。

默认规格：

- 宽度 `72px`，小视口 `56px`。
- 背景 `--navia-brand-soft`。
- avatar 44px 圆形按钮。
- online cue 使用 `--navia-success`；offline 使用 `--navia-error` 或明确离线状态。

### 5.4 ChatWorkspace

职责：

- 展示 header、当前页面信息、快捷动作、消息流、输入区。
- 消息区独立滚动。
- 输入区固定在底部。
- 接收并呈现 SSE 增量。

布局：

```text
grid-template-rows:
  header
  page context
  quick actions
  minmax(0, 1fr) message list
  composer
```

消息规则：

| Role | 视觉 |
|---|---|
| user | 右侧或突出显示，使用 `--navia-user-bg` |
| assistant | 左侧/全宽，白色或轻背景 |
| system | 辅助样式，使用 `--navia-system-bg` |

验收：

- 长消息不撑破面板。
- 新回复 streaming 时不重排输入区。
- 用户在底部时自动跟随新消息；用户滚动查看历史时不得强制跳到底部，除非当前实现无法低风险实现，则记录为 V1.2 候选。

### 5.5 ToolDock

职责：

- 右侧竖向工具入口。
- V1.1 默认只做视觉占位，不新增工具能力。

入口：

```text
聊天 / 画图 / 视频 / 音频 / 发现
```

规则：

- `聊天` 为 active。
- 未实现入口必须呈现 disabled 或 placeholder 语义，不触发 Runtime 工具。
- 不新增 API。

### 5.6 ArtifactViewer

职责：

- 展示 Mermaid iframe。
- 展示 source fallback。
- 展示渲染失败状态。

规则：

- iframe 使用 `mermaid-renderer.html`。
- `metadata.format === "mermaid"` 才进入 Mermaid viewer。
- render failure 展开 source details。
- Artifact 区不得让消息列表横向溢出。

## 6. State Matrix

| 状态 | 必须可见内容 | 禁止 |
|---|---|---|
| Runtime checking | 状态文本或 cue | 空白面板 |
| Runtime offline | 离线提示、重连入口 | 允许发送正常 chat |
| PageContext missing | 读取当前页面入口 | 生成假摘要 |
| Page submitted | 当前页面 title/domain/headings | 丢失 active page |
| Streaming | assistant message 增量、send disabled | 重复创建 session |
| Tool running | tool 状态文本或轻提示 | UI 冻结 |
| Tool failure | 错误消息和可继续输入状态 | 无限重试 |
| Mermaid failure | 后续 mindmap / Mermaid 专项定义 | 在 V1.1-B/C 中误声明 mindmap 视觉完成 |
| Session restored | 恢复页面标题和历史消息 | 只恢复前端本地假状态 |

## 7. File-Level Implementation Plan

### 7.1 `src/injectedPanel.ts`

必须做：

- 将 `styles()` 顶部改为 token-first CSS。
- 将 `markup()` 分区重命名为目标语义 class，同时保留现有 `data-testid`。
- 增强 `render()` 对 `data-runtime`、`data-submitted`、`data-stream` 的同步，供 CSS 状态选择器使用。
- 为 error / empty / streaming / artifact 状态补充稳定 DOM 容器。

不得做：

- 不把 AgentCore 状态写入前端。
- 不绕过 `runtimeClient`。
- 不删除现有 Mermaid iframe fallback。

### 7.2 `src/panelLayout.ts`

必须保持：

- `MIN_PANEL_WIDTH = 440`。
- `MOBILE_OVERLAY_BREAKPOINT = 900`。
- `OVERLAY_ENTER_RATIO = 0.52`。
- `PUSH_RETURN_RATIO = 0.48`。
- `MAX_PANEL_RATIO = 0.8`。

只有在 Figma 或人类确认后才能调整这些阈值，并同步更新 PRD / acceptance / tests。

### 7.3 Tests

必须扩展：

- `injectedPanel.test.ts`：检查目标语义 class、状态 dataset、artifact fallback、按钮 disabled。

建议新增：

- `chrome-inpage-visual-e2e.mjs`：只负责视觉状态截图和证据输出。

## 9. Manual Review Checklist

人工视觉审查必须回答：

- [ ] 浮动球尺寸、位置、阴影、贴边方向是否符合设计。
- [ ] Hover 小长条是否从正确方向伸出，动画终态是否自然。
- [ ] `440px` 面板内左轨、聊天区、右工具区比例是否符合设计。
- [ ] 半屏状态下消息区和输入区是否稳定。
- [ ] 覆盖态是否符合设计对底层网页可交互性的要求；如未确认，是否采用不可交互背景。
- [ ] 小视口降级是否可用且不遮挡输入区。
- [ ] Runtime offline、PageContext missing、tool failure、Mermaid fallback 是否有高保真样式。
- [ ] 视觉重构后 summary / QA / mindmap / session restore 是否仍可用。
