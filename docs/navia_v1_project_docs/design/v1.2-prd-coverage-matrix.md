# V1.2 PRD 覆盖矩阵

版本：V1.2 PRD Coverage Matrix

## 1. 使用方式

每个 V1.2 模块完成开发后，必须在本矩阵中填入实际 evidence 路径。没有 evidence 的需求不能声明完成。

证据填写规则：

- A/B/C/D 模块阶段只填写 module evidence。
- E Integration 阶段必须把真实 E2E evidence 路径补入本矩阵。
- 任何只包含 mock 文本、没有 fixture 名称、没有 assertion log 或没有 trace 的证据，不能声明对应 PRD 完成。
- PRD A-F 页面内交互不回退由 E 阶段最终确认。

## 3. No-Go

- evidence 只包含 mock 文本，不能证明合同。
- B evidence 不能证明 SSE 事件来源。
- C evidence 没有 `nodeSourceMap`。
- D evidence 没有 `turnId` / `toolCallId`。
- Integration evidence 缺少真实 Chrome 或真实 fixture。
- PRD evidence 路径仍是 placeholder 时声明 V1.2 complete。
