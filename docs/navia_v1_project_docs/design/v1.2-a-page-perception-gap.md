## 1. 目标

本图谱专门服务 A 模块开发。A 模块是 `Page Perception / AgentCore Eyes`，负责把网页与未来媒体环境中的可读事实转成高密度、可追踪、可被 D/C 消费的结构化上下文。

本图谱只冻结开发方向、合同、验收和出门条件，不代表已实现 OCR、视觉模型、视频识别或直播识别。

## 3. A 模块目标架构口径

- A 输入 `PageReadingInput`。
- A 输出 `PageReadingResult` 和 `StructuredPageContext`。
- A 只负责结构化感知、段落标注、chunk、heading path、信息密度和 source reference。
- A 不负责 ChatBox turn 编排、最终回答、artifact、SSE、mindmap 生成或 UI 渲染。
- A 不直接调用 MCP、Skill、外部 API、OCR engine、视觉模型、视频流分析或直播流分析。
- 未来 OCR / 视觉 / 媒体识别能力必须通过 D Adapter Layer 和 governance hooks 接入。

## 6. No-Go

- A 直接生成 summary / answer / mindmap artifact。
- A 直接发 SSE 或写 EventStore。
- A 直接调用 D/C/B、MCP、Skill、外部 API。
- A 直接调用 OCR engine、视觉模型、视频流分析或直播流分析。
- A 在没有 alt/caption/title/aria/nearby text 或已批准 OCR 输入时声称理解图片内容。
- A 将 `A-V2.0-*` 视频 / 直播规划声明为 V1.2 已完成能力。

## 7. A-V1.1 目标架构补充

A-V1.1 在 `StructuredPageContext` 之上增加高信号视图：

```text
StructuredPageContext
-> CandidateExtractionResult
-> A-owned block graph
-> HighSignalPageContext
-> SourceMap / SourceRef
-> PerceptionDigest
-> PagePerceptionQualityReport
```

调用关系：

```text
A Page Reading
-> HighSignalPageContext / PerceptionDigest / SourceMap / QualityReport
-> D CoreProvider context when quality readiness pass
-> C Mindmap source grounding
-> B Debug evidence and future jumpback fallback
```

关键门槛：

- HighSignal / Digest / SourceMap / QualityReport 是公共合同后，D/C 才能依赖 exact shape。
- `SourceRef` 必须有 fallback text，DOM selector 只是可选增强。
- `PagePerceptionQualityReport` 必须由公式计算，不能写死 pass。
- Candidate extractor 输出不能直接暴露给 D/C/B。
- A-V1.1-1+ 在 A-V1.1-0a 到 0e 审计闭环前 No-Go。

## 8. A-V1.2 生产级网页感知补充

A-V1.2 的目标是从当前 DOM baseline 升级为复杂真实网页可验收的生产级网页感知规划。

推荐组合路线：

```text
DOM baseline enhancement
+ third-party extractor candidate ensemble
+ gold fixture / 100-page quality evaluation
```

目标架构：

```text
PageReadingInput
-> dom_baseline candidate
-> trafilatura candidate
-> readability_lxml candidate
-> readabilipy candidate
-> CandidateExtractionResult[]
-> A-owned block graph merge
-> HighSignalPageContext
-> SourceMap / SourceRef
-> PerceptionDigest
-> PagePerceptionQualityReport
-> ExtractorComparisonReport
```

100-page 验收：

- 至少 100 个复杂真实网页或可复现 HTML snapshot。
- 至少 10 个类别，每个核心类别有独立 pass rate。
- 每个页面必须有 category、complexityTags、expectedRisks、URL 或 snapshotPath。
- 每个页面必须产出 structured / candidate / high-signal / source-map / digest / quality / comparison evidence。
- valid-content pass rate 必须 `>= 85%`。
- 任一核心类别 pass rate `< 70%` 时不得出门。
- low-signal / paywall-like 页面必须 fail/degrade，不能 pass。

No-Go：

- 少于 100 个网页就声明 production-grade。
- 第三方 extractor 原始输出泄漏到 Navia 公共合同。
- 未做依赖、license、性能、fallback 审计就接入 extractor。
- 默认执行 OCR / VLM / ASR / video / live engine。
- 为了通过大规模验收放宽 A-V1.1 核心 SourceRef / QualityGate。
