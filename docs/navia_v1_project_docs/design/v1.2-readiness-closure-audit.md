当前 V1.2 文档已经补齐 readiness closure、A 专项 drawio、V1.2 总图谱、PRD coverage、Integration matrix，以及 V1.2-B/C/D/E 可执行门禁。文档水平可以支撑后续全部 V1.2 staged mock-first 自动化开发，但仍必须先完成 ChatGPT 审计，并在审计无致命或重大规格偏差后进入实质开发。

本轮补强后的目标结论是：

```text

No-Go for real piAgentProvider implementation until piAgent dependency lock is complete.
```

## 3. V1.2-0R 必须闭环的 Gap

| Gap | 风险 | 闭环方式 |
|---|---|---|
| B/C/D/E stage gate 偏大纲 | 独立 Codex 开发时自行决策 | 在各 stage gate 补开发顺序、fixture、证据、No-Go |
| PRD coverage evidence 仍是 placeholder | false green | E 阶段必须填真实 evidence 路径 |
| D/piAgent 适配边界不够硬 | Core 绕过治理或 trace | real piAgentProvider 保持 No-Go，直到 dependency lock |
| B/C source fallback 链路不够明确 | mindmap 视觉可用但不可追踪 | C 产出 `nodeSourceMap`，B 消费 source map 或 excerpt fallback |
| Integration 可能重写模块业务逻辑 | 模块隔离失效 | Integration 只 wiring，不重写 A/B/C/D |

## 4. 剩余开发计划大纲

### V1.2-0R：Readiness Closure

目标：

- 冻结 V1.2 全量开发审计文档清单。
- 补齐 B/C/D/E stage gate 的开发与验收粒度。
- 确认 A 专项 drawio、V1.2 总 drawio、PRD coverage、Integration matrix 互相一致。

验收：

- `git diff --check` 通过。
- V1.2 关键文档清单少于 20 个。
- B/C/D/E stage gate 均包含开发顺序、fixture、evidence、No-Go 和 PRD 复检。
- `v1.2-ai-reading-automation-gap.md` 和 `v1.2-a-page-perception-gap.md` 均能回答架构、合同、开发、验收和出门条件。

### V1.2-A：Page Perception / AgentCore Eyes

目标：

- 从真实 HTML fixture / PageContext 生成 `StructuredPageContext`。
- 输出 stable paragraph/chunk IDs、annotations、contentHash 和 optional summary draft。
- OCR、视频、直播只保留合同规划。

验收：

- article/docs/github_readme/image-rich/table-list-code/empty fixtures 通过。
- empty page 返回 `PAGE_CONTEXT_REQUIRED`。
- 不创建 artifact，不发 SSE，不调用 D/C/B/MCP/Skill/API。

### V1.2-D：CoreProvider + Adapter Layer

目标：

- 先实现 `MockCoreProvider` 合同。
- 实现 CoreTurnInput/CoreTurnResult、AdapterRegistry、GovernanceBridge、ToolResult/Event/Artifact/Trace mapping。
- piAgentProvider 仅冻结适配合同。

验收：

- summary、QA、mindmap、denied adapter、missing activePage、budget exceeded fixtures 通过。
- denied / budget exceeded 在 `tool.started` 前阻断。
- trace 可按 `turn_id` 还原 state、intent、budget、tool、artifact、response、error。
- real piAgentProvider 不得进入实现，除非 dependency lock 已闭环。

### V1.2-C：Mindmap Service

目标：

- 从 `StructuredPageContext` 生成 Mermaid source。
- 生成 `nodeSourceMap`。
- validation + at-most-once repair。

验收：

- Mermaid source 可校验。
- repairCount `<= 1`。
- 主要节点可追踪 paragraph/chunk。
- C 不创建 UI，不发 SSE，不自行抽取网页。

### V1.2-B：Frontend Renderer

目标：

- 只消费 SSE、ArtifactRecord、RuntimeStatus。
- 渲染 streaming text、structured summary、tool state、artifact、mindmap、debug。
- unknown event 和 failure states 不崩溃。

验收：

- recorded SSE replay 生成稳定 view model。
- offline、missing context、tool failure、Mermaid render failure 可见。
- B 不直接调用 A/C/D/MCP/Skill/API。

### V1.2-E：Integration / E2E / PRD Review

目标：

- 接通 A -> D -> C -> B。
- 使用真实 Chrome + Runtime + 真实网页完成端到端验收。
- 填写 PRD coverage matrix evidence 路径。

验收：

- 真实网页读取、总结、追问、mindmap、source fallback 和 trace 均通过。
- PRD A-F 页面内交互不回退。
- Runtime tests、extension tests、typecheck、Chrome E2E、真实手工验收均有记录。
