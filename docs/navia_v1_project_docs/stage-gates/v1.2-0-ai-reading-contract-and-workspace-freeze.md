## 1. 阶段目标

V1.2-0 的目标是在任何实质代码开发前冻结以下内容：

- A/B/C/D 四模块职责。
- A/C/D service 工作区和 B app 工作区。
- Integration Codex 的职责。
- `StructuredPageContext` 合同。
- Adapter 合同。
- Mindmap 反跳 source map 合同。
- 跨模块合同变更流程。

本阶段完成后，后续四个 Codex 终端可以分别开发 A/B/C/D，并只在自己的工作区内修改代码和验收。

---

## 2. PRD 规格检视

V1 产品体验仍以 `PRD/窗口交互_PRD.md` 为准：

```text
页面边缘悬浮球
-> hover 小长条
-> 网页内 AI 双轨聊天面板
-> 窄距/半屏挤压
-> 宽工作区覆盖
-> 收起恢复
```

V1.2 扩展只深化“聊天”页签的 AI 伴读能力，不改变悬浮球、面板布局和 V1.1 前端高保真口径。

V1.2 扩展允许：

- 轻量 MCP / Skill / API Adapter 合同。
- 单 Session 连续上下文。
- 结构化网页 JSON。
- Mindmap 节点回跳来源。

V1.2 扩展不允许：

- 长期记忆。
- RAG。
- 多 Agent。
- 浏览器自动操作。
- 默认本地文件读取。
- 高风险外部 side effect 自动执行。

## 7. false-green 风险

| 风险 | 级别 | 防线 |
|---|---|---|
| 只写模块名但没有目录边界 | 重大 | `workspace-partition` 必须列出允许工作区 |
| MCP / Skill 绕过 D | 致命 | Adapter 合同规定只能通过 D |
| B 直接生成回答或 Mindmap | 致命 | B 只消费 SSE 和 Artifact |
| D 变成长期记忆 / RAG | 致命 | V1.2 只允许单 Session 连续上下文 |
| 每个 Codex 终端跨目录乱改 | 重大 | 跨模块变更必须回到 V1.2-0 |
| C Mindmap 无法反跳来源 | 重大 | `MindmapNodeSourceMap` 必须写入 artifact metadata |
