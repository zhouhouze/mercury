## 4. 验收标准

- Unit tests 通过。
- Typecheck 通过。
- Build 通过。
- 真实 Chrome in-page E2E 通过。
- required states 均有可检查状态或截图证据：floating default、floating hover、panel 440 push、panel 50vw push、panel overlay、mobile overlay、runtime offline、PageContext missing、tool failure。
- 长文本、长消息、Artifact 不撑破布局。
- 输入区固定，消息区独立滚动。
- Runtime offline、PageContext missing、tool failure 不空白，不生成假摘要/假回答，并提供恢复路径。
- Mindmap artifact 不作为高保真视觉通过项。

## 5. 真实数据验收

必须跑真实 Chrome 页面内交互：

- Runtime online。
- Runtime offline。
- PageContext submit。
- Summary / QA。
- Mindmap artifact existence only；Mermaid / mindmap 视觉专项后续处理。

正式命令：

```bash
node scripts/validate_v1_1_doc_readiness.mjs
pnpm --dir apps/chrome-extension test
pnpm --dir apps/chrome-extension typecheck
pnpm --dir apps/chrome-extension build
NAVIA_E2E_HEADLESS=false pnpm --dir apps/chrome-extension e2e:inpage
```

补充：

- `NAVIA_E2E_HEADLESS=false` 是当前真实扩展 E2E 的固定参数，因为 headless Chromium 在本机环境无法稳定加载 MV3 unpacked extension。
- 如新增 visual E2E 截图脚本，应输出到 `apps/chrome-extension/e2e/screenshots/current/`，不得混入 Figma baseline 目录。
