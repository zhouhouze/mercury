# V1.2-D CoreProvider / Adapter Workspace 门禁

版本：V1.2-D Stage Gate Draft
日期：2026-06-04
负责模块：D，CoreProvider 适配层、ChatBox turn 编排边界与 MCP / Skill / API Adapter 编排

---

## 1. 阶段目标

将 D 模块定义为 Navia Runtime 与可替换 Agent Core Provider 之间的稳定适配层。V1.2 首选 Core Provider 为 piAgent；D 负责 CoreProvider 抽象、AdapterRegistry、GovernanceBridge、ToolResult / ArtifactRecord / AgentEvent / Trace 映射，并继续保证 state、budget、permission、SSE 和 Trace 可审计。

---

## 2. 允许工作区

D 模块 Codex 只允许修改：

```text
services/local-runtime/navia_runtime/modules/agent_loop/
services/local-runtime/navia_runtime/modules/adapters/
docs/navia_v1_project_docs/stage-gates/v1.2-d-agentic-loop.md
```

禁止修改 B UI 实现、A/C 输出合同、公共 API 合同、`agent.py` 和 `tools.py` 既有入口。需要接入既有 Runtime 时由 Integration Codex wiring；需要变更合同时回到 V1.2-0。

---

## 4. 验收标准

- [ ] 每个 user message 创建 `turn_id`。
- [ ] 使用 fake adapters 和 activePage fixtures 生成 turn event evidence。
- [ ] 每个 adapter call 创建 `tool_call_id`。
- [ ] Budget / Permission check 在 `tool.started` 前执行。
- [ ] denied adapter 不产生 `tool.started`。
- [ ] MCP / Skill / API 不绕过 D。
- [ ] piAgentProvider 不直接写 ArtifactRecord、SSE、EventStore 或 UI。
- [ ] `NAVIA_AGENT_CORE_PROVIDER=mock | piagent | custom` 的配置策略已在文档中闭环。
- [ ] trace 可按 `turn_id` 看到 state、intent、budget、tool、artifact、response、error。
- [ ] D 不实现长期记忆、RAG、多 Agent 或浏览器自动操作。

---

## 5. 预审计

当前结论：

```text
Ready for ChatGPT audit after V1.2-0 final documentation alignment.
Go for mock-first CoreProvider / Adapter Layer implementation only after audit closes all fatal/major issues.
No-Go for real piAgentProvider implementation until piAgent repository/version/license lock is complete.
```

No-Go：

- D 默认执行高风险 MCP / Skill side effect。
- D 引入长期记忆或 RAG。
- D 允许前端直连 adapter。
- D 允许 AdapterResult 跳过 ToolResult envelope。
- piAgent 或其他 CoreProvider 绕过 D Adapter Layer 直接写结果。
- piAgent 具体仓库、版本或 license 未锁定就进入实质接入。
