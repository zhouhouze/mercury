# V1.2-A / A-V1.2 Production Page Perception 门禁

版本：A-V1.2 Planning Draft
日期：2026-06-05
负责模块：A，Page Perception / AgentCore Eyes
状态：Documentation planning only

## 1. 阶段定位

A-V1.2 收敛为网页感知层阶段：

```text
高质量网页感知
+ 结构化页面摘要
+ 可反跳证据
+ Debug 可验证 JSON
```

A 模块只负责把当前网页转成高密度、可验证、可反跳的页面事实输入。A 不生成最终回答，不生成 Mindmap，不生成学习卡片、Quiz、Podcast、Notebook，不创建 Artifact，不发 SSE，不写 EventStore / Trace，不直接调用 D/C/B、MCP、Skill、外部 API、OCR、VLM、ASR、视频或直播 engine。

## 2. 推荐技术路线

默认路线：

```text
DOM baseline
+ candidate extractor ensemble
+ A-owned schema normalization
+ SourceMap / jumpback
+ Quality Evaluator
+ DebugEvidenceBundle
+ 100-page corpus gate
```

路线含义：

- `DOM baseline` 是最低实现和离线 fallback。
- `candidate extractor ensemble` 只提供候选抽取结果，必须通过依赖审计后才可启用。
- `A-owned schema normalization` 是 D/C/B 唯一可依赖的输出归一化层。
- `SourceMap / jumpback` 让摘要项、关键事实和高信号块都能回到来源。
- `Quality Evaluator` 必须用可计算指标决定 `pass / degraded / fail`，不得写死通过。
- `DebugEvidenceBundle` 是开发者和验收方判断 A 是否读懂网页的证据包。
- `100-page corpus gate` 是 A-V1.2 完成声明的硬门槛。

备选路线：

| 路线 | A-V1.2 定位 | 优点 | 风险 |
|---|---|---|---|
| DOM heuristic only | 必做 baseline | 可控、无外部依赖、合同稳定 | 复杂网页鲁棒性有限 |
| Readability 系路线 | candidate extractor | 适合文章页，社区成熟 | 对文档、表格、代码、动态网页弱 |
| trafilatura 路线 | candidate extractor | 新闻、博客、论文页效果较好 | Python 依赖、license、性能和输出隔离需审计 |
| Ensemble 路线 | 推荐主路线 | 质量上限高，适合 100-page 验收 | 需要候选比较、去重和 fallback 逻辑 |
| LLM-as-extractor | 不默认实装 | 语义表达质量可能高 | 成本、隐私、不可复现、幻觉风险 |
| Browser snapshot / multimodal | V1.3+ 规划 | 可支持复杂视觉页面 | 工程复杂，涉及 OCR/VLM/ASR 边界 |

## 3. 子阶段开发与验收计划

| 阶段 | 开发目标 | 出门条件 |
|---|---|---|
| `A-V1.2-0` | 合同与目标冻结 | PRD、架构、公共合同、drawio、100-page 规则、DebugEvidenceBundle 口径一致；审计无致命或重大风险 |
| `A-V1.2-1` | 100-page 验收集 | 至少 100 个复杂真实网页的可复现 snapshot；每个最终计入页面有 category、complexityTags、expectedRisks、URL 和 snapshotPath |
| `A-V1.2-2` | 主体内容识别 | 正文、标题层级、段落、列表、表格、代码、图片 DOM metadata 可进入 A-owned block graph |
| `A-V1.2-3` | 噪声过滤与密度提升 | nav/footer/sidebar/recommendation/ad/comment/share/cookie/banner 可过滤或降权，并输出 filtered evidence |
| `A-V1.2-4` | 结构化页面摘要 | 输出 TLDR、关键段落、关键事实、术语、实体、步骤、表格事实、代码事实；每项必须有关联 sourceRefs |
| `A-V1.2-5` | SourceMap 与反跳证据 | SourceRef 包含 pageId、contentHash、blockId、blockType、order、textQuote、textHash、fallbackText、confidence；selector/domPath/textOffset 仅为可选增强 |
| `A-V1.2-6` | 质量评估模型 | noiseRatio、contentCoverage、sourceCoverage、groundingCompleteness、jumpbackCoverage、digestCompressionRatio、overallScore 均有公式和阈值 |
| `A-V1.2-7` | Debug JSON 证据包 | DebugEvidenceBundle 包含 raw signals、candidate extraction、filtered evidence、high-signal context、source map、quality report |
| `A-V1.2-8` | 100-page 出门验收 | `eval_corpus` 读取 `corpus-manifest.json` 并输出 `corpus-level-report.json`；真实 corpus 验收通过，失败则以非零退出并打回对应子阶段重新审计 |

## 4. 100-page 验收矩阵

最低要求：

```text
total_pages >= 100
category_count >= 10
each_core_category >= 8 pages
final counted pages have URL and snapshotPath / reproducible HTML evidence
final counted pages have goldStatus reviewed or semi_auto_accepted
all pages have category, complexityTags, expectedRisks, and evidence outputs
```

出门命令：

```bash
PYTHONPATH=services/local-runtime python3 -m navia_runtime.modules.page_reading.eval_corpus \
  --manifest services/local-runtime/navia_runtime/modules/page_reading/tests/evidence/a_v1_2/corpus-manifest.json \
  --output services/local-runtime/navia_runtime/modules/page_reading/tests/evidence/a_v1_2/corpus-level-report.json
```

命令必须在类别门槛、source coverage、grounding completeness、jumpback coverage、低信号页、gold review、snapshot 可复现性或 debug evidence 任一失败时以非零退出。

推荐类别：

| 类别 | 最低数量 | 主要风险 |
|---|---:|---|
| `news_article` | 8 | 广告、推荐、侧栏、发布时间噪声 |
| `longform_blog` | 8 | 长文、脚注、作者 bio、订阅区 |
| `technical_docs` | 8 | 多级标题、代码、API 表 |
| `github_readme` | 8 | markdown 转 HTML、列表、代码块、徽章噪声 |
| `product_docs` | 8 | nav、版本切换、侧边目录 |
| `ecommerce_product` | 8 | 价格、规格、评论、推荐商品 |
| `forum_thread` | 8 | 多用户回复、引用、低信号评论 |
| `academic_or_report` | 8 | 表格、引用、章节、PDF-like HTML |
| `table_heavy_page` | 8 | 表头、单元格、跨行结构 |
| `code_heavy_page` | 8 | 代码块、命令、错误日志 |
| `image_rich_article` | 8 | alt/caption 缺失、图注、图库噪声 |
| `localized_chinese_page` | 8 | 中文断句、站点导航、混合语言 |
| `low_signal_or_paywall_like` | 4 | 登录墙、付费墙、空内容、cookie banner |

每个样本必须产出或规划产出：

```text
structured-page.json
high-signal-page.json
source-map.json
perception-digest.json
quality-report.json
debug-evidence.json
```

## 5. 质量门槛

通过页最低门槛：

```text
sourceCoverage >= 0.95
groundingCompleteness >= 0.95
jumpbackCoverage >= 0.90
noiseRatio <= 0.25 or degraded with explicit reason
PerceptionDigestItem.sourceRefs.length > 0
DebugEvidenceBundle explains pass / degraded / fail
```

质量报告必须包含：

```text
metric.value
metric.numerator
metric.denominator
metric.method
metric.threshold
metric.passed
```

低信号、登录墙、付费墙、空内容页面必须 `fail` 或 `degraded`，不得为了整体通过率被标记为 `pass`。

## 7. No-Go

以下情况必须停止：

- 少于 100 个复杂网页就声明 A-V1.2 完成。
- 未冻结 100-page corpus 规则就实现 extractor ensemble。
- 新增 extractor 依赖但未做 license / size / runtime / fallback 审计。
- 第三方 extractor 输出直接成为最终 Navia 合同。
- A 生成最终回答、Mindmap、Flashcards、Quiz、Podcast、Notebook 或 Artifact。
- A 发 SSE、写 EventStore、写 Trace、调用 D/C/B、MCP、Skill 或外部 API。
- A 默认执行 OCR、VLM、ASR、视频或直播 engine。
- `quality-report.json` 写死 pass 或放宽 A-V1.2 high-signal 兼容 SourceRef / QualityGate。
- Debug 页能显示 JSON，但 JSON 不能解释 pass / degraded / fail 原因。
