# V1.2-A / A-V1.2 Production-Grade Page Perception 门禁

版本：A-V1.2 Planning Draft
日期：2026-06-05
负责模块：A，Page Perception / AgentCore Eyes
状态：Documentation planning only

## 1. 阶段目标

A-V1.2 的目标是把 A 模块从 `DOM baseline high-signal perception` 升级为面向复杂真实网页的生产级感知规划。

本阶段解决：

- 复杂网页正文识别质量不足。
- 导航、侧栏、推荐、广告、评论、重复链接等噪声仍可能污染输出。
- `PerceptionDigest` 仍偏 deterministic 截断，缺少真实网页质量对照。
- `PagePerceptionQualityReport` 已有合同级指标，但缺少大规模真实网页校准。
- D/C/B 下游需要更稳定、更高密度、更可反跳的网页事实输入。

本阶段必须新增至少 `100` 个复杂不同种类网页的验收规划，并在后续实现阶段用这些网页或其可复现快照完成质量验收。

## 2. 推荐组合技术路线

默认路线：

```text
DOM baseline enhancement
+ third-party extractor candidate ensemble
+ gold fixture / 100-page quality evaluation
```

保留但不默认实装：

```text
LLM-assisted digest refinement
OCR / VLM / ASR / video / live perception
```

路线判断：

| 路线 | A-V1.2 定位 | 价值 | 风险 |
|---|---|---|---|
| DOM baseline enhancement | 必做基础层 | 离线、可控、继承当前实现 | 复杂网页上限有限 |
| Extractor candidate ensemble | 主路线 | 提升真实网页正文识别质量 | 新依赖、输出隔离、license / 性能审计 |
| Gold fixture quality evaluation | 必做验收层 | 防止质量 false green | 需要 100-page corpus 和 gold rubric |
| LLM-assisted digest refinement | 规划 / mock only | 提升 digest 表达质量 | 成本、隐私、可复现性、A 越界风险 |
| Multimodal planning | 规划 only | 为 OCR / 视频 / 直播铺路 | 过早实装会扩大隐私和治理风险 |

## 4. 100 个复杂网页验收要求

最低要求：

```text
total_pages >= 100
category_count >= 10
each_core_category >= 8 pages
all pages have URL or reproducible HTML snapshot
all pages have category, complexity tags, expected risks, and evidence outputs
```

推荐类别：

| 类别 | 最低数量 | 主要风险 |
|---|---:|---|
| `news_article` | 8 | 广告、推荐、侧栏、发布时间噪声 |
| `longform_blog` | 8 | 长文、脚注、作者 bio、订阅区 |
| `technical_docs` | 8 | 多级标题、代码、API 表 |
| `github_readme` | 8 | markdown 转 HTML、列表、代码块、徽章噪声 |
| `product_docs` | 8 | nav、版本切换、侧边目录 |
| `ecommerce_product` | 8 | 价格、规格、评论、推荐商品 |
| `search_result` | 8 | 结果列表、摘要片段、广告 |
| `forum_thread` | 8 | 多用户回复、引用、低信号评论 |
| `academic_or_report` | 8 | 表格、引用、章节、PDF-like HTML |
| `table_heavy_page` | 8 | 表头、单元格、跨行结构 |
| `code_heavy_page` | 8 | 代码块、命令、错误日志 |
| `image_rich_article` | 8 | alt/caption 缺失、图注、图库噪声 |
| `multi_column_media_page` | 4 | 多栏布局、播放器壳、推荐视频 |
| `localized_chinese_page` | 4 | 中文断句、站点导航、混合语言 |
| `low_signal_or_paywall_like` | 4 | 登录墙、付费墙、空内容、cookie banner |

每个样本必须产出或规划产出：

```text
structured-page.json
candidate-extraction.json
high-signal-page.json
source-map.json
perception-digest.json
quality-report.json
comparison-report.json
```

## 5. 验收门槛

A-V1.2 完成后才能声明 production-grade page perception planning ready，最低门槛：

```text
valid_content pass rate >= 85% across 100-page corpus
sourceCoverage >= 0.95 for passed pages
groundingCompleteness >= 0.95 for passed pages
jumpbackCoverage >= 0.95 for passed pages
no_signal / low_signal pages must fail or degrade, never pass
third-party extractor raw output must not leak into final public contracts
comparison-report must identify winning candidate and fallback reason
```

类别维度门槛：

- 每个核心类别必须有独立 pass rate。
- 任一核心类别 pass rate `< 70%` 时，A-V1.2 不得整体出门。
- `low_signal_or_paywall_like` 的目标不是 pass，而是正确 fail/degrade。
- `multi_column_media_page` 只能验证页面文本和媒体壳，不声明视频理解。

## 6. 公共合同策略

A-V1.2 默认复用 A-V1.1 公共合同：

```text
HighSignalPageContext
SourceMap / SourceRef
PerceptionDigest

```

允许新增的规划字段必须先进入 `A-V1.2-0` 合同冻结：

- `ExtractorComparisonReport`
- `ExtractorCandidateScore`
- `GoldEvaluationRecord`
- `CorpusPageRecord`

若 D/C/B 需要依赖新字段 exact shape，必须更新 `contracts/v1_2_adapter_contracts.md` 或新增公共 schema，并重新审计。

## 7. No-Go

以下情况必须停止，不得进入 A-V1.2-1+ 实现：

- 未冻结 100-page corpus 规则就实现 extractor ensemble。
- 新增 `trafilatura`、`readability-lxml`、`readabilipy` 依赖但未做 license / size / runtime / fallback 审计。
- 第三方 extractor 输出直接成为最终 Navia 合同。
- 只用 5-10 个手写 HTML fixture 声明生产级网页感知。
- quality report 为了通过 100-page 验收而放宽 A-V1.1 核心门槛。
- A 调用 D/C/B、Artifact、SSE、EventStore、MCP、Skill、外部 API。
- A 默认执行 OCR、VLM、ASR、视频或直播 engine。
- 把多模态 planning 声明为 A-V1.2 已实现能力。
