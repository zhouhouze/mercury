## 5. 端到端验收

真实数据主链路：

```text
打开真实网页
-> 读取网页
-> A 生成 StructuredPageContext
-> D 创建 turn
-> D 调用 summary adapter
-> B 流式渲染 summary
-> 用户追问
-> D 使用单 Session 历史和 activePage 回答
-> D 调用 C mindmap adapter
-> B 渲染 mindmap
-> 点击节点反跳 source paragraph 或展示 excerpt fallback
-> trace?turn_id 可审计
```

必须通过：

- [ ] Runtime tests。
- [ ] Extension unit tests。
- [ ] Typecheck。
- [ ] Chrome in-page E2E。
- [ ] 真实 Chrome 手工验收。
- [ ] PRD A-F 交互不回退。
- [ ] 更新 `design/v1.2-prd-coverage-matrix.md` 中实际 evidence 路径。
- [ ] trace by `turn_id` 可审计 state、intent、budget、tool、artifact、response、error。
- [ ] artifact source chain 包含 sourcePageId、turnId、toolCallId 和 source paragraph/chunk。
- [ ] missing activePage、tool failure、Mermaid render failure 均在 UI 可见。

真实数据场景：

- [ ] 普通文章网页：读取、总结、追问、mindmap。
- [ ] 技术文档网页：heading/chunk/source map。
- [ ] GitHub README 风格网页：列表/代码块和 source fallback。
- [ ] empty / blocked readable content：`PAGE_CONTEXT_REQUIRED`。
