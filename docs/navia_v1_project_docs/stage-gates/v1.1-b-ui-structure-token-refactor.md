## 1. 阶段目标

在不改变功能链路的前提下，将注入式前端结构整理为稳定语义分区，并把样式集中到 visual tokens。

---

## 2. PRD 规格检视

必须保持：

- 页面内悬浮球为主入口。
- hover 小长条为主要打开热区。
- 网页内双轨面板。
- push / overlay / resize / collapse 行为不回退。
- Side Panel 只作为调试入口。

---

## 3. 开发计划

允许改动：

- `injectedPanel.ts` 中 markup 语义分区。
- `injectedPanel.ts` 中 CSS token 化。
- 如风险可控，可拆出 `injectedPanelMarkup.ts`、`injectedPanelStyles.ts`、`injectedPanelViewModel.ts`。

必须保留：

- 现有 `data-testid`。
- Shadow DOM 隔离。
- `runtimeClient` 调用路径。
- Mermaid iframe fallback。
- `panelLayout.ts` 阈值。
