## 1. 阶段目标

V1.0-G 的目标是将 V1 单 Session 历史从内存基线升级为可恢复、可追踪的本地 SQLite 持久化底座。

本阶段支撑 V1 complete 的硬门槛：

- Side Panel 刷新后 Session 不丢。
- Session / Turn / Tool / Artifact / Event / Trace 可审计。
- EventLog 来自持久 EventStore，不只是实时 EventStream。

---

## 2. PRD 规格检视

### 对齐目标

- V1 不做长期记忆，但必须把单 Session 做实。
- Chrome Side Panel 只缓存 `session_id`，AgentCore 状态仍归 Runtime。
- V2 Memory Plane 的升级路径依赖 V1 PageContext、Artifact 和 EventLog。

### 非目标

本阶段不做：

- RAG。
- 长期记忆。
- Memory Plane。
- 多 Agent。
- 云同步。
- 账号系统。
- 本地文件读取。

---

## 3. 开发计划

### G.1 SQLite Runtime Store

实现内容：

- SQLite EventStore。
- SQLite SessionStore。
- Session / Page / Message / ToolCall / Artifact / BudgetLedger / Checkpoint 表。
- 默认数据库路径 `.navia/navia.sqlite3`。
- `NAVIA_DB_PATH` 支持测试和自定义路径。

验收：

- Runtime 重启后可重新打开同一 SQLite 数据库。
- `trace` 从 SQLite EventStore 读取。

### G.2 Session Restore API

实现内容：

- `GET /v1/sessions/{session_id}`。
- 返回 session summary、activePage、messages、artifacts、toolCalls、budgetLedger、checkpoints。
- activePage 不返回完整正文，避免普通恢复 API 泄漏网页全文。

验收：

- 不存在的 session 返回 `SESSION_NOT_FOUND` envelope。
- 已完成 chat 的 session 可恢复 turn 记录。

### G.3 Side Panel Refresh Recovery

实现内容：

- Side Panel 使用 `chrome.storage.local` 缓存最近 `session_id`。
- Runtime online 后调用 `GET /v1/sessions/{session_id}` 恢复视图。
- 恢复 activePage、messages 和 Mermaid artifact 展示。

验收：

- 前端不保存 AgentCore 核心状态。
- Runtime 返回 404 时清理本地缓存。
