版本：V1.0-E Gate Draft
日期：2026-06-01
阶段状态：实质开发与自动验收已完成；等待真实 Chrome Side Panel 人工验收

## 1. 阶段目标

V1.0-E 的目标是在已经通过手工验收的 Chrome Side Panel + PageContext 链路上，交付 V1 的基础网页伴读能力：

```text
当前网页 PageContext
-> Side Panel Chatbox
-> /v1/chat/stream SSE
-> AgentCore
-> 网页摘要 / 页面问答 / 选区解释 / Mermaid mindmap
-> UI 展示
-> trace 可审计
```

完成 V1.0-E 后，用户应能在 Chrome 插件里基于当前网页完成基础文字对话。

---

## 2. PRD 规格检视

### 2.1 对应 PRD 目标

- 提供一个 Chatbox。
- 支持网页伴读。
- 根据当前浏览网页进行内容识别、提取和总结。
- 生成 Mermaid 思维导图。
- 对话、工具调用、状态迁移和结果可审计。

### 2.2 本阶段必须满足的 V1 complete 目标

- Side Panel 可完成文字对话。
- 当前网页 `PageContext` 被 Runtime 工具真实消费。
- 支持“总结这篇文章”。
- 支持“基于当前页面回答问题”。
- 支持生成 Mermaid mindmap artifact。
- 每个 chat turn 有 `turn_id`。
- 每个 tool call 关联 `session_id`、`turn_id`。
- 每个 artifact 有 `sourcePageId`、`turnId`、`toolCallId`。
- 默认不读取本地文件，不联网搜索，不执行浏览器自动操作。

### 2.3 本阶段非目标

V1.0-E 不做：

- RAG / 长期记忆。
- MCP / Skill。
- 多 Agent。
- 本地文件读取。
- 浏览器自动操作。
- YouTube 页面内嵌。
- 网页贴边悬浮球。
- 语音输入。
- Session 跨重启恢复质量优化。
- 深度研究或 PPT 生成。

## 3. 目标架构摘要

完整架构见：

```text
docs/navia_v1_project_docs/design/v1.0-e-reading-tools-architecture.md
```

核心架构原则：

- 前端只做交互与展示，AgentCore 状态以 Runtime 为准。
- `/v1/chat/stream` 是 E 阶段唯一主对话入口。
- 摘要、问答、选区解释、mindmap 均以工具形式执行。
- 工具统一返回 `ToolResult` envelope。
- 工具结果统一沉淀为 `ArtifactRecord`。
- 所有工具调用都必须经过 governance hooks。
- 所有状态迁移、工具调用、artifact 创建都必须写入 EventStore。
- 缺少 activePage 时统一使用既有 `PAGE_CONTEXT_REQUIRED`。
- 不新增 ad-hoc event string。

## 4. 开发计划

### E.1 Runtime PageContext 与 Artifact 底座

目标：

- 让 Runtime 工具可以读取 session `activePage` 的真实 PageContext。
- 补齐 `ArtifactRecord` 最小结构和事件。

任务：

- 扩展 `SessionStore` 保存完整最小 PageContext：
  - `page_id`
  - `url`
  - `title`
  - `domain`
  - `content_hash`
  - `captured_at`
  - `headings`
  - `cleaned_text`
  - `selected_text`
  - `chunks`
- 新增或补齐 `ArtifactRecord`：
  - `artifact_id`
  - `type`
  - `metadata.format`
  - `sourcePageId`
  - `sourceChunkIds`
  - `turnId`
  - `toolCallId`
  - `content`
- 新增 `artifact.created` event。
- `GET /v1/sessions/{session_id}/trace?turn_id=...` 可看到 artifact 事件。
- ArtifactRecord 不新增顶层 `format`，格式统一写入 `metadata.format`。
- `explain_selection` 产物使用 `ArtifactRecord(type="answer", source="selection")`，不新增 `type="explanation"`。

验收：

- PageContext 提交后，chat 工具可读取真实 `cleaned_text`。
- 每个工具成功后产生 artifact。
- artifact 与 turn/tool/source page 关联完整。

### E.2 IntentRouter 扩展

目标：

- 识别网页伴读核心意图。

任务：

- 支持摘要意图：
  - “总结”
  - “概括”
  - “这篇文章讲什么”
- 支持页面问答意图。
- 支持选区解释意图。
- 支持 mindmap 意图：
  - “生成思维导图”
  - “mindmap”
  - “mermaid”
- 未识别意图返回可恢复说明，不产生虚假工具成功。

验收：

- 固定用例可路由到正确工具。
- 未知问题不会误触高风险工具。

### E.3 网页伴读工具

目标：

- 工具不再 mock，必须消费真实 PageContext。

任务：

- 实现 `summarize_page` deterministic baseline。
- 实现 `answer_from_page` deterministic baseline。
- 实现 `explain_selection`。
- 实现 `generate_mindmap` deterministic baseline。
- 实现 Mermaid validator。
- 实现 Mermaid repair once 或明确失败。
- Mermaid validation / repair 详情写入 `tool.done` data 或 artifact metadata，不新增 AgentEvent 类型。

验收：

- 对真实 article fixture 可生成摘要。
- 对真实 docs fixture 可回答页面相关问题。
- 对真实 github README fixture 可生成 mindmap。
- 没有 activePage 时返回 `PAGE_CONTEXT_REQUIRED`，不得生成假摘要、假回答或假 artifact。
- `read_local_file` 仍默认 deny。

### E.4 Side Panel Chatbox 与 SSE

目标：

- Side Panel 可进行真实文字对话。

任务：

- 新增聊天消息列表。
- 新增输入框和发送按钮。
- 消费 `/v1/chat/stream` SSE。
- 展示 streaming response。
- 展示 tool running / done 状态。
- 展示 error 状态。
- 未知 SSE event 不得导致 UI 崩溃。
- Runtime offline、PageContext required、tool failure、Mermaid render failure 必须可见。
- 添加快捷动作：
  - 总结
  - 问答
  - Mindmap
  - 解释选区
- PageContext 未提交时，提示用户先读取并提交页面。

验收：

- Runtime online 时可完成一轮 chat。
- Runtime offline 时不出现空白或静默失败。
- PageContext missing 时不产生 false green。

### E.5 Mermaid 展示

目标：

- 用户可以看到 mindmap artifact。

任务：

- 确认 Mermaid 渲染策略。
- 安装并接入前端 `mermaid` 包。
- Runtime 生成并验证 Mermaid source。
- Frontend 真实渲染 Mermaid。
- Mermaid render error 时展示 source fallback 和错误状态。
- 如果真实渲染未实现，则 V1.0-E 只能声明 Mermaid source artifact，不得声明 Mermaid visual acceptance 或 V1 complete。

验收：

- Mermaid source 通过 validator。
- Side Panel 中可看到真实 Mermaid 渲染结果。
- 若渲染失败，UI 展示明确错误和 source fallback，不无限重试。

### E.6 E2E 验收与 PRD 复检

目标：

- 使用真实数据完成端到端验收。

任务：

- Runtime 自动测试。
- Chrome extension 单元测试。
- 前端 build smoke。
- 真实 fixture HTTP 验收。
- Chrome Side Panel 手工或自动 E2E。
- PRD 规格复检。

验收：

- 所有测试通过。
- 至少一个真实 Chrome tab 完成摘要、问答、mindmap。
- trace 可按 `turn_id` 还原主路径。

- [ ] 每个 turn 有 `turn_id`。
- [ ] 每个 tool call 有 `tool_call_id`、`session_id`、`turn_id`。
- [ ] 每个 artifact 有 `sourcePageId`、`turnId`、`toolCallId`。
- [ ] `trace?turn_id=...` 可看到同一 turn 的 state、intent、tool、artifact、response 事件。
- [ ] `PAGE_CONTEXT_REQUIRED` 场景不会生成假结果。
- [ ] `read_local_file` deny 场景不会产生 `tool.started`。

### 5.2 真实数据验收

### 5.3 前端验收

- [ ] Runtime offline 时 UI 显示 offline 和重连入口。
- [ ] Runtime online 时 UI 显示 online。
- [ ] PageContext 未提交时，Chatbox 不假装已拥有页面上下文。
- [ ] 用户可以发送消息。
- [ ] 用户可以点击“总结”快捷动作。
- [ ] 用户可以点击“Mindmap”快捷动作。
- [ ] streaming 期间 UI 有明确状态。
- [ ] 工具失败时 UI 显示错误。
- [ ] Mermaid artifact 有真实渲染展示。
- [ ] Mermaid render failure 时显示 source fallback。
- [ ] 未知 SSE event 不导致 UI 崩溃。

### 5.4 false-green 防线

以下任一情况出现，V1.0-E 不得通过：

- 工具返回仍是 mock 文本，未读取 activePage。
- Chatbox 前端显示成功，但 trace 中没有对应 turn。
- 有 tool result，但没有 ToolResult envelope。
- 有 artifact 展示，但没有 `artifact.created` event。
- Mermaid 不可验证仍声明成功。
- PageContext missing 时仍返回“总结成功”。
- Missing activePage 未使用 `PAGE_CONTEXT_REQUIRED`。
- Mermaid 只展示 source 却声明 visual acceptance 或 V1 complete。
- Budget 或 permission check 未出现在 tool.started 前。
- 前端保存 AgentCore 核心状态。
- 默认开启本地文件读取。

## 9. 实施摘要

已完成 V1.0-E 实质开发。

- `SessionStore` 支持读取完整 `activePage`。

  - 总结
  - Mindmap
