# V1.2-A / A-V1.1 High-Signal Page Perception 门禁

版本：V1.2-A / A-V1.1 Stage Gate Draft
日期：2026-06-05
负责模块：A，Page Perception / AgentCore Eyes

## 1. 阶段目标

A-V1.1 把 A 模块从“结构化抓取”升级为“高信号、可评估、可反跳、可供 D/C 直接消费的网页感知层”。

本阶段解决四类问题：

- JSON 噪声过多，主内容被淹没。
- 关键信息提取和蒸馏不足。
- 缺少自动质量评估，仍依赖人工判读。
- 图片 / 视频 / 反跳能力不足。

推荐路线：

```text
Hybrid Extraction Pipeline
= open-source article extraction baseline
+ A-owned DOM block / sourceMap
+ deterministic perception digest
+ quality evaluator
+ governed multimodal adapter contracts
```

## 2. 允许工作区

A-V1.1 默认只允许修改：

```text
services/local-runtime/navia_runtime/modules/page_reading/
docs/navia_v1_project_docs/stage-gates/v1.2-a-page-reading.md

docs/navia_v1_project_docs/design/v1.2-a-page-perception-gap.md
docs/navia_v1_project_docs/MODULE_VERSIONING.md
```

禁止事项：

- A 直接生成最终 assistant answer。
- A 发 SSE、写 EventStore、创建 Artifact。
- A 直接调用 D/C/B、MCP、Skill 或外部 API。
- A 直接下载图片、上传图片、调用 OCR/VLM/ASR/视频/直播 engine。
- A 在没有 source reference 时把 digest item 标记为高可信。

如需修改公共合同，必须先回到 `A-V1.1-0` 并同步项目级合同文档。

## 3. 开发计划

审计结论：

```text
Go for A-V1.1-0 contract freeze.
No-Go for A-V1.1-1+ implementation until P0 contract gaps are closed.
```

| 阶段 | 目标 | 验收重点 |
|---|---|---|
| `A-V1.1-0` | 冻结 `HighSignalPageContext`、`PerceptionDigest`、`PagePerceptionQualityReport`、`SourceMap` module-local 合同 | 第三方抽取库只作为 candidate；公共合同变更必须回到冻结流程 |
| `A-V1.1-1` | 正文识别与噪声过滤 | 主内容进入 high-signal 输出；nav/footer/aside/recommendation/ad-like/comment 降权或过滤 |

| `A-V1.1-5` | 图片与 OCR 合同增强 | 图片只基于 DOM metadata 或受控 OCR mock；不幻想图片内容 |
| `A-V1.1-6` | 视频 / 直播感知规划登记 | 只冻结合同和验收口径，不实现真实 engine |

|---|---|---|

### 3.2 Public vs Module-Local 决策

默认决策：

```text
Promote to public contract before D/C consumption.
```

理由：

- A-V1.1 目标是让 D/C 直接消费高信号上下文和 source reference。
- 若保持 module-local，D/C/B 不得依赖 exact shape，只能在 Debug 中展示 evidence。
- 因此 A-V1.1-0 必须把 HighSignalPageContext、PerceptionDigest、SourceMap / SourceRef 和 PagePerceptionQualityReport 提升到 `contracts/v1_2_adapter_contracts.md` 或等价公共合同。

未完成公共合同升级前：

```text
A-V1.1 outputs are evidence only.
D/C/B must not depend on their exact shape.
A-V1.1-1+ implementation is No-Go.
```

### 3.3 A-V1.1-0 Schema Freeze

必须冻结以下精确 schema，并提供 JSON Schema 或 Pydantic model 与 validation tests：

```text
HighSignalPageContext
HighSignalBlock
FilteredBlockEvidence
PerceptionDigest
PerceptionDigestItem
SourceMap
SourceRef
PagePerceptionQualityReport
QualityMetric
QualityIssue
CandidateExtractionResult
CandidateBlock
```

推荐最低结构：

```text
HighSignalPageContext
- schemaVersion
- pageId
- sessionId
- contentHash
- sourceStructuredPageRef { pageId, contentHash }
- metadata
- highSignalBlocks[]
- filteredBlocks[]
- sourceMapRef
- digestRef?
- qualityReportRef
- status: ready | degraded | failed
- warnings[]

PerceptionDigest
- digestId
- pageId
- contentHash
- items[]
- summary { tldr, keyTakeaways[] }
- stats { itemCount, sourceRefCount, compressionRatio }

PerceptionDigestItem
- itemId
- kind: key_fact | entity | claim | evidence | definition | procedure | open_question | table_fact | code_fact | image_metadata
- text
- importance
- confidence
- sourceRefs[]
- relatedParagraphIds[]
- relatedChunkIds[]
- warnings[]

SourceMap
- sourceMapId
- pageId
- contentHash
- sourceRefs[]

SourceRef
- sourceRefId
- pageId
- contentHash
- blockId
- blockType: paragraph | chunk | heading | image | table | table_cell | code | list_item | quote
- order
- paragraphId?
- chunkId?
- headingPath?
- textQuote
- textHash
- selector?
- domPath?
- startOffset?
- endOffset?
- fallbackText
- confidence
```

硬规则：

- 每个 `PerceptionDigestItem` 必须有 `sourceRefs`。
- 没有 `sourceRefs` 的候选 item 只能进入 `warnings` 或 `rejectedItems`，不得进入 `digest.items`。
- `selector` / `domPath` 不得作为唯一反跳机制。
- 每个 `SourceRef` 必须有 `textQuote` 或 `fallbackText`。

### 3.4 A-V1.1-0 Quality Metric Formula Freeze

`PagePerceptionQualityReport` 必须采用可审计 metric 结构：

```text
QualityMetric
- value
- numerator?
- denominator?
- method
- threshold?
- passed
```

必须冻结以下公式：

| 指标 | 公式口径 |
|---|---|
| `noiseRatio` | filteredOrDowngradedNoiseBlocks / allDetectedBlocks |
| `contentCoverage` | highSignalContentChars / readableContentChars |
| `sourceCoverage` | highSignalBlocksWithSourceRef / highSignalBlocksTotal |
| `groundingCompleteness` | digestItemsWithSourceRefs / digestItemsTotal |
| `jumpbackCoverage` | sourceRefsWithTextQuoteOrFallbackText / sourceRefsTotal |
| `digestCompressionRatio` | digestTextTokenEstimate / structuredPageTextTokenEstimate |
| `candidateFactDensity` | digestCandidateFactItems / digestTokenEstimate |
| `overallScore` | weighted deterministic score from passed metrics; weights frozen in schema docs |

默认门槛：

```text
overallScore >= 0.75
sourceCoverage >= 0.95
groundingCompleteness >= 0.95
jumpbackCoverage >= 0.95
noiseRatio <= 0.25
downstreamReadiness = pass
```

禁止：

- `QualityReport` 写死 pass。
- 未提供 numerator / denominator / method 的指标参与 overallScore。
- 将 `candidateFactDensity` 声明为事实真伪验证。

### 3.5 A-V1.1-0 Candidate Extractor Isolation

第三方 extractor 输出必须进入隔离结构：

```text
CandidateExtractionResult
- extractorName: trafilatura | readabilipy | readability_lxml | dom_baseline
- version?
- title?
- text?
- metadata?
- blocks?
- confidence?
- warnings[]
```

规则：

- `CandidateExtractionResult` 不得直接返回给 D/C/B。
- `CandidateExtractionResult` 不得直接写入 final HighSignalPageContext 字段。
- A Pipeline 必须把 candidate output 映射回 A-owned block graph 和 SourceMap。
- 若第三方库不可用，必须 fallback 到 DOM baseline。

### 3.6 A-V1.1-0 Fixture Class Gates

fixture 必须分组验收：

| 分组 | 示例 | 预期 |
|---|---|---|
| `valid_content` | article_noise、news_with_sidebar、product_doc、table_heavy_report、code_doc | 必须 `downstreamReadiness = pass` |
| `degraded_content` | image_rich_article 中无 DOM metadata 的局部图片内容 | 允许 degraded，但必须说明原因 |
| `no_signal` | empty_or_low_signal | 必须 fail 或 `PAGE_CONTEXT_REQUIRED`，不得 pass |
| `planning_only` | video_page_stub | 只验证合同字段，不得标记 real perception ready |

## 4. 技术选型

正文抽取候选层：

- 首选：`trafilatura`，用于正文与 metadata 降噪 baseline。
- 对照：Mozilla Readability / `readabilipy`，用于验证正文抽取质量。
- fallback：`readability-lxml`，用于轻量场景。

约束：

- 第三方库输出是候选输入，不是 Navia 最终事实。
- 最终 `StructuredPageContext`、`HighSignalPageContext`、`PerceptionDigest` 由 A Pipeline 统一生成。
- 任一第三方库不可绕过 sourceMap、quality report 和 false-green gate。

## 5. 验收标准

必须新增或扩展真实 HTML fixture：

```text
article_noise.html
news_with_sidebar.html
product_doc.html
image_rich_article.html
table_heavy_report.html
code_doc.html
video_page_stub.html
empty_or_low_signal.html
```

每个有效 fixture 必须生成：

```text
*.structured-page.json
*.high-signal-page.json
*.perception-digest.json
*.quality-report.json
*.source-map.json
```

质量门槛：

```text
overallScore >= 0.75
sourceCoverage >= 0.95
groundingCompleteness >= 0.95
noiseRatio <= 0.25
downstreamReadiness = pass
```

硬验收：

- high-signal 输出不能只是 `cleanedText` 的复制。
- digest item 必须有 source reference。
- 被过滤内容必须有 debug evidence。
- 无 DOM metadata 或 OCR mock 的图片必须输出 unknown，不得描述图像内容。
- 视频 / 直播只能输出规划合同或 stub evidence，不得声明真实识别完成。
- `empty_or_low_signal.html` 必须 fail 或 degraded，不得 pass。
- `video_page_stub.html` 必须 planning-only，不得 real perception ready。
- D/C/B 在公共合同升级前不得依赖 A-V1.1 exact shape。

## 7. PRD 规格复检

Pass 条件：

- A 只交付感知事实、source map、digest 和质量评估。
- A 不交付最终回答、前端体验、Artifact、SSE 或 AgenticLoop。
- A 的输出能降低噪声并提升 D/C 输入质量。
- A 的质量报告能让开发者不依赖人工肉眼判断基础质量。

No-Go 条件：

- 把第三方正文抽取库输出直接当作最终合同。
- digest 无来源。
- 质量报告只写死通过。
- 用 mock 数据声明真实页面验收通过。
- 把 OCR、VLM、视频或直播 engine 声明为 A-V1.1 已完成能力。
- 未完成 A-V1.1-0a 到 A-V1.1-0e 就进入 A-V1.1-1+ 实现。
