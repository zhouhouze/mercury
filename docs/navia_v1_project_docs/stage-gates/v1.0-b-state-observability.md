# V1.0-B Stage Gate：StateMachine & Observability

版本：V1.0-B Gate Baseline
日期：2026-06-01
阶段状态：V1.0-B StateMachine & Observability 已完成，允许启动 V1.0-C 阶段门禁准备

---

## 1. 阶段目标

V1.0-B 的目标是把状态机和可观测能力从 V1.0-A 的主路径最小实现，提升为正式 runtime contract：

```text
TransitionTable
-> StateMachine validation
-> INVALID_TRANSITION
-> AgentEvent schema validation
-> Mermaid state graph
-> Trace export
-> turn_id filter
```

---

## 2. PRD 规格检视

### 2.1 对应 PRD 目标

- Agent 状态机可视化、可验证、可观测、可扩展。
- 每次状态迁移写入 EventStore。
- `/v1/agent/state-machine/mermaid` 输出状态图。
- `/v1/sessions/{session_id}/trace` 输出 Session 追踪。

### 2.2 必须保留的 V1 complete 目标

- Side Panel 后续可展示 Agent 状态。
- 当前网页文字对话的每个 turn 都可 trace。
- 工具调用和响应流可被审计。

### 2.3 本阶段非目标

V1.0-B 不做：

- Chrome 插件 UI。
- 完整预算治理。
- SQLite 持久化。
- 真实模型。
- Mindmap 工具。
- RAG / Memory / Multi-Agent。

---

## 3. 开发计划

### 3.1 StateMachine

- 扩展 TransitionTable 为唯一状态迁移来源。
- 增加 `INVALID_TRANSITION` error mapping。
- 补主路径和异常路径状态。
- TurnRunner 必须继续通过 `StateMachine.transition()`。

### 3.2 Observability

- 增加 `GET /v1/agent/state`。
- 完善 `GET /v1/agent/state-machine/mermaid`。
- 完善 `GET /v1/sessions/{session_id}/trace`。
- 支持 trace 按 `turn_id` 过滤。
- 对所有输出事件做 AgentEvent schema validation 测试。

### 3.3 Mermaid Snapshot

- TransitionTable 生成 Mermaid。
- Snapshot 测试确保图和 transition table 一致。

---

## 4. 验收标准

- [ ] Transition table 是唯一真实来源。
- [ ] 非法 transition 返回或抛出 `INVALID_TRANSITION`。
- [ ] 每次 transition 写入 EventStore。
- [ ] Mermaid 图由 TransitionTable 生成。
- [ ] `/v1/agent/state` 返回当前状态。
- [ ] `/v1/agent/state-machine/mermaid` 返回 envelope + Mermaid。
- [ ] `/v1/sessions/{session_id}/trace` 可返回完整 trace。
- [ ] Trace 可按 `turn_id` 过滤。
- [ ] AgentEvent schema validation 覆盖 chat stream 所有事件。
- [ ] 未引入 V1 非目标。
