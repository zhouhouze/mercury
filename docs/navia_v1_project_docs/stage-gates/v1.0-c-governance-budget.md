# V1.0-C Stage Gate：Governance / Budget Supervisor

版本：V1.0-C Gate Baseline  
日期：2026-06-01  
阶段状态：V1.0-C Governance / Budget Supervisor 已完成，允许启动 V1.0-D 阶段门禁准备

---

## 1. 阶段目标

V1.0-C 的目标是把 V1.0-A 的空治理 hook 升级为可执行的最小治理底座：

```text
TurnBudget
-> BudgetSupervisor
-> PermissionSupervisor
-> ContextSupervisor
-> FileQuerySupervisor 默认 deny
-> ApprovalGate 最小事件
-> BudgetLedger / ToolResult.budget_cost
```

---

## 2. PRD 规格检视

### 2.1 对应 PRD 目标

- Agent 具备预算、权限、上下文和本地文件访问监督机制。
- 工具调用前必须经过 policy check。
- 本地文件默认不读取。
- 超预算后不继续工具调用。

### 2.2 必须保留的 V1 complete 目标

- Chrome Side Panel 后续可展示预算状态。
- 当前网页问答链路仍可跑通。
- Trace 中能看到 budget / permission 相关事件。
- 工具调用必须可审计。

### 2.3 本阶段非目标

V1.0-C 不做：

- 完整审批 UI。
- 本地文件读取 allowlist。
- Shell 执行。
- 浏览器自动操作。
- 精确 token 计费。
- 多 Agent。
- RAG / Memory Plane。

---

## 3. 开发计划

### 3.1 BudgetSupervisor

- 定义 TurnBudget。
- 支持 `max_tool_calls`。
- 支持 `max_retries`。
- 支持 `max_context_bytes`。
- 超预算前不得产生 `tool.started`。
- 写入 budget checked / budget exhausted event。

### 3.2 PermissionSupervisor

- 默认 allow：
  - `summarize_page`
  - `answer_from_page`
  - `explain_selection`
  - `generate_mindmap`
- 默认 deny：
  - `read_local_file`
  - `search_local_workspace`
  - `shell`
  - `browser_automation`

### 3.3 ApprovalGate

- 对 high-risk tool 只产生 `approval_required` 或 `tool.denied` 事件。
- 审批前不得执行 side effect。
- V1.0-C 不做完整 approve/reject UI。

### 3.4 ToolExecutor 集成

- ToolExecutor 必须在 `tool.started` 前完成 budget 和 permission 检查。
- deny / budget_exceeded 时不得调用工具函数。
- ToolCallRecord status 不得进入 `running`。

## 9. 实施摘要

已完成。

新增或增强：

- `TurnBudget`：支持 `max_tool_calls`、`max_retries`、`max_context_bytes`。
- `BudgetExceeded`：工具预算耗尽时阻断工具执行。
- `GovernanceHooks.pre_tool_use()`：在 `tool.started` 前执行 permission 和 budget check。
- `ToolExecutor.authorize()`：将授权检查和实际工具执行拆分，确保 deny 时不会调用工具函数。
- RuleBasedIntentRouter 识别本地文件请求并路由为 `read_local_file`。
- TurnRunner 在 permission denied / budget exceeded 时只产生 `tool.denied` 和解释性 response，不产生 `tool.started`。

本阶段未实现：

- 完整审批 UI。
- 本地文件读取 allowlist。
- Shell。
- 浏览器自动操作。
- 精确 token 计费。
