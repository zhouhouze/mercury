# V1.0-A Stage Gate：AgentCore Baseline

版本：V1.0-A Gate Baseline  
日期：2026-06-01  
阶段状态：V1.0-A AgentCore Baseline 已完成，允许启动 V1.0-B 阶段门禁准备

---

## 1. 阶段目标

V1.0-A 的目标是在 V1.0-0 合同与 Runtime Skeleton 之上，实现一轮最小可审计 AgentCore loop：

```text
user message
-> turn_id
-> rule-based intent
-> PreToolUse
-> mock tool
-> ToolResult
-> PostToolUse
-> response.delta / response.done
-> EventStore trace
```

本阶段只做单 Session、单轮最小闭环，不做 Chrome 插件、不做真实模型、不做 RAG、不做长期记忆。

---

## 2. PRD 规格检视

### 2.1 对应 PRD 目标

本阶段对应总 PRD 中的：

- 单 Session 高质量对话。
- AgentCore 使用状态机运行。
- 本地意图识别先用 rule-based。
- 工具调用必须经过预算、权限和上下文监督 hook。
- 每轮对话可通过 EventStore / Trace 审计。

### 2.2 必须保留的 V1 complete 目标

V1.0-A 不实现 Chrome UI，但必须支撑最终 V1 complete：

- `/v1/chat/stream` 可被 Side Panel 消费。
- 每次文字输入创建 `turn_id`。
- 每次工具调用创建 `tool_call_id`。
- 每个工具返回 ToolResult envelope。
- Trace 可还原 turn 主路径。

### 2.3 本阶段非目标

V1.0-A 不做：

- Chrome extension。
- Content Script。
- `/v1/page/context` 正式链路。
- 真实 LLM。
- Mindmap 真实生成。
- SQLite 持久化。
- 完整 BudgetSupervisor。
- 完整异常路径。
- MCP / Skill / RAG / Multi-Agent / browser automation。

## 3. 开发计划

### 3.1 Runtime 模块

- 保留 V1.0-0 API envelope、SSE、EventStore/EventStream。
- `POST /v1/chat/stream` 改为调用 `TurnRunner`，不得在 endpoint 中手写业务 loop。
- Trace 继续从 EventStore 读取。

### 3.2 AgentCore 模块

新增最小模块：

- `IntentRouter`：rule-based，识别 summarize / ask / unknown。
- `ToolRegistry`：注册 `summarize_page`、`answer_from_page`。
- `ToolExecutor`：统一生成 ToolCallRecord 和 ToolResult。
- `GovernanceHooks`：PreToolUse / PostToolUse 空实现。
- `TurnRunner`：通过 StateMachine.transition() 执行主路径。

### 3.3 Mock 工具

`summarize_page`：

- 输入：message、page_id、fixture-derived PageContext metadata optional。
- 输出：ToolResult envelope。
- 必须创建 summary artifact id。

`answer_from_page`：

- 输入：message、page_id。
- 输出：ToolResult envelope。
- 当前信息不足时返回说明，不联网、不读本地文件。

### 3.4 事件要求

一轮 chat 至少产生：

```text
state.transition
intent.detected
budget.checked
tool.started
tool.done
response.delta
response.done
```

## 4. 验收标准

- [ ] 一轮 chat 产生 `turn_id`。
- [ ] IntentRouter 能识别“总结这篇文章”。
- [ ] IntentRouter 能识别基于页面的问题。
- [ ] ToolExecutor 不绕过 PreToolUse。
- [ ] ToolExecutor 不绕过 PostToolUse。
- [ ] `summarize_page` 返回 ToolResult envelope。
- [ ] `answer_from_page` 返回 ToolResult envelope。
- [ ] 每个 tool call 关联 `session_id` 和 `turn_id`。
- [ ] SSE 输出包含 `intent.detected`、`tool.started`、`tool.done`。
- [ ] Trace API 能看到同一个 turn 的完整事件。
- [ ] 高风险工具仍默认不可用。
- [ ] 未引入 Chrome 插件、RAG、多 Agent 或本地文件读取。
