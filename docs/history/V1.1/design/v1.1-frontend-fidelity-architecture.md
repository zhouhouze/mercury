# V1.1 前端体验高保真目标架构

版本：V1.1 Frontend Fidelity Architecture
日期：2026-06-03
适用阶段：V1.1 Frontend Fidelity / Figma Visual Alignment

---

## 1. 阶段定位

V1.0 的目标是完成“功能闭环 + 页面内悬浮球交互骨架”：

```text
当前网页
-> 页面内悬浮球 / hover 小长条 / 网页内双轨面板
-> PageContext
-> Local Runtime
-> SSE Chat
-> 摘要 / 问答 / Mermaid
-> Session / Trace / Governance
```

V1.1 的目标不是新增 Agent 能力，而是把这个可用骨架升级为可对照 Figma 原型验收的高保真前端体验。

实现级细节以 `v1.1-frontend-fidelity-implementation-spec.md` 为准，包括 visual token、组件结构、状态矩阵、文件级改动计划、截图验收脚本规格和人工视觉审查清单。

V1.1 不改变：

- Runtime API。
- AgentEvent。
- ToolResult。
- PageContext。
- Session / Turn / Tool / Artifact / Event 数据合同。
- Governance / Permission / Budget 边界。

V1.1 只改变：

- 页面内注入 UI 的结构语义。
- 视觉 token 和状态样式。
- 高保真截图验收流程。
- Figma 原型到真实注入面板的映射文档。

---

## 2. 当前架构

当前 V1.0 页面内体验主链路：

```text
Chrome Content Script
-> mountNaviaInjectedPanel()
-> Shadow DOM host
-> injectedPanel markup/styles/render loop
-> Runtime API client
-> PageContext / SSE / Session restore / Mermaid iframe
```

当前实现优势：

- 已与真实网页共存，具备 Shadow DOM 样式隔离。
- 已实现悬浮球、hover 小长条、展开/收起、resize、push / overlay 阈值。
- 已打通 PageContext、Runtime、SSE Chat、Mermaid artifact、Session restore。
- 已有布局规则与 E2E 验收脚本基础。

当前实现差距：

- 面板结构偏工程模板，缺少清晰组件语义。
- CSS token 分散，难以和 Figma 视觉规格逐项对齐。
- 左轨、聊天主区、右工具区和 artifact 区缺少产品级层级。
- 错误态、空态、streaming、tool running、fallback 等状态只满足可用，不满足高保真。
- 缺少截图基线和 Figma 对照验收。

## 3. 目标架构

Figma Make 暴露出的目标组件语义：

```text
App
-> MainLayout
-> MockPage
-> FloatingBall
-> Sidebar
-> ChatArea
```

V1.1 真实注入面板映射：

| Figma 语义 | 真实注入面板目标 | 说明 |
|---|---|---|
| `MainLayout` | `InjectedPanelShell` | 控制整体固定定位、左右停靠、push / overlay、resize、收起恢复 |
| `MockPage` | `PageContextPreview` / E2E fixture | 只用于预览和验收语境，不替代真实网页 |
| `FloatingBall` | `FloatingEntry` | 默认态、hover 态、小长条、吸附边、拖拽记忆 |
| `Sidebar` | `LeftRail` + `ToolDock` | 左轨 avatar / collapse，右侧工具入口 |
| `ChatArea` | `ChatWorkspace` | 消息列表、快捷动作、输入区、streaming、artifact |

V1.1 Interface Plane：

```text
Floating Entry
  - default
  - hover strip
  - drag / dock memory

Panel Shell
  - left / right dock
  - push mode
  - overlay mode
  - resize handle
  - collapse recovery

Left Rail
  - avatar / collapse
  - runtime status cue

Chat Workspace
  - page context header
  - quick actions
  - message timeline
  - fixed composer
  - independent scroll

Tool Dock
  - chat / diagram / video / audio / discover placeholders

Artifact Viewer
  - Mermaid iframe render
  - source fallback
  - validation failure state

Visual Tokens
  - color
  - typography
  - spacing
  - radius
  - shadow
  - motion
  - track widths
```

## 4. 数据与事件流

V1.1 不新增数据流。目标是保证视觉重构后以下链路不回退：

```text
Open panel
-> checkRuntimeHealth()
-> extractPageContext()
-> submitRuntimePageContext()
-> streamRuntimeChat()
-> handle AgentEvent
-> render message / artifact / status
-> restoreRuntimeSession()
```

前端状态只允许保存：

- open / closed。
- dock side。
- y position。
- panel width。
- runtimeStatus。
- sessionId cache。
- pageContext display copy。
- messages display copy。
- streamStatus。

前端不得保存：

- AgentCore 核心状态。
- Tool execution authority。
- Budget authority。
- 长期知识库。
- 本地文件索引。

## 5. 视觉验收基线

V1.1 必须在进入最终验收前具备至少一种视觉基线：

- Figma 原型截图；或
- 普通 Figma `/design` 节点链接；或
- 人类确认的静态视觉参考图。

仅有 Figma Make 资源清单时，可以进入结构和 token 重构，但不能声明“视觉高保真通过”。

必须截图验收的状态：

| 状态 | 视口 | 验收重点 |
|---|---|---|
| 默认浮动球 | 1280x900 | 贴边、阴影、尺寸、不遮挡内容 |
| Hover 小长条 | 1280x900 | 伸出方向、高亮、快捷提示、动画终态 |
| 窄距展开 | 1280x900 | `440px` 面板、网页挤压、左轨/聊天区/工具区比例 |
| 半屏展开 | 1280x900 | `50vw` 附近布局稳定、聊天区滚动 |
| 覆盖态 | 1280x900 | `>52vw` 后不继续挤压，最大 `80vw` |
| 小视口 | 390x844 或 800x900 | 覆盖式或全屏侧栏降级 |
| 错误态 | 1280x900 | Runtime offline、PageContext missing、tool failure |
| Artifact | 1280x900 | Mermaid SVG 或 source fallback |

## 6. 出门条件

V1.1 可以声明 ready 仅当：

- Figma 对照基线存在。
- PRD A-F 状态真实 Chrome 复验通过。
- Playwright 截图基线与当前截图记录齐全。
- Runtime / PageContext / SSE / Mermaid / Session restore 全链路不回退。
- `v1.1-frontend-fidelity-gap.drawio` 可打开，且完整反映当前架构、目标架构、差异、开发及验收计划、里程碑、验收门槛和出门条件。

V1.1 不得声明 ready 如果：

- 只能 Side Panel 通过。
- 只有 DOM 测试，没有截图验收。
- 没有 Figma 或人工视觉基线。
- 截图明显偏离目标布局比例。
- 面板关闭后网页布局无法恢复。
- Runtime / Trace / Session 链路断开。
