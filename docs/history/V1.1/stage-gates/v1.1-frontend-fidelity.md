# V1.1 Stage Gate：前端体验高保真

版本：V1.1 Frontend Fidelity Gate
日期：2026-06-03
阶段状态：文档计划已创建；未进入实质前端开发

---

## 1. 阶段目标

V1.1 的目标是把 V1.0 已可用的页面内悬浮球与网页内 AI 双轨面板，升级为可对照 Figma 原型验收的高保真前端体验。

本阶段必须保持：

- V1.0 PageContext 链路不回退。
- Runtime / AgentCore / Governance / Trace 仍为状态源。
- `/v1/chat/stream` SSE 消费不回退。
- Mermaid artifact 渲染或 source fallback 不回退。
- Session restore 不回退。
- PRD A-F 布局状态不回退。

## 3. 开发计划

### V1.1-A：Figma 视觉规格冻结

- 使用 Chrome CLI 对 Figma Make live preview 进行截图硬切，或收集普通 Figma `/design` 节点。
- 截图输出到 `design/v1.1-figma-baseline/current/`，登录/权限失败截图不得作为基线。
- 将人工复核后的截图登记并沉淀到 `design/v1.1-figma-baseline/reviewed/`。
- 从 Figma Make 资源清单确认组件语义：`MainLayout / MockPage / FloatingBall / Sidebar / ChatArea`。
- 形成视觉 token 草案：颜色、字体、间距、圆角、阴影、轨道宽度、面板宽度、动画。
- 明确覆盖态下底层网页是否可交互；如无设计确认，默认不可交互。
- 独立门禁文档：`stage-gates/v1.1-a-visual-baseline-freeze.md`。

### V1.1-B：前端结构与 token 重构

- 将注入面板结构映射为 Floating Entry、Panel Shell、Left Rail、Chat Workspace、Tool Dock、Artifact Viewer。
- 集中 token，避免分散魔法值。
- 保留现有 `data-testid` 或建立兼容映射，避免 E2E 断裂。
- 保留 Shadow DOM 或同等 CSS 隔离策略。
- 独立门禁文档：`stage-gates/v1.1-b-ui-structure-token-refactor.md`。

### V1.1-C：高保真样式实现与状态补齐

- 重做默认态、hover 态、展开态、半屏态、覆盖态、小视口态样式。
- 补齐 Runtime offline、PageContext missing、tool running、tool failure、streaming 状态；Mermaid / mindmap 后续专项。
- 优化聊天区独立滚动、固定输入区、消息密度和 artifact 卡片。
- 保持页面挤压与收起恢复行为。
- 独立门禁文档：`stage-gates/v1.1-c-high-fidelity-states.md`。

### V1.1-D：截图基线与真实 Chrome 复验

- 新增或更新 Playwright 截图脚本。
- 截图覆盖默认态、hover 态、窄距、半屏、覆盖、小视口、错误态、artifact。
- 真实 Chrome 普通网页复验 PRD A-F。
- 记录截图路径、环境、浏览器版本、fixture 来源。
- 独立门禁文档：`stage-gates/v1.1-d-visual-e2e-regression.md`。

### V1.1-E：文档收口与出门评审

- 更新 V1.1 stage-gate 验收报告。
- 更新 gap drawio。
- 更新 README / onboarding map / acceptance plan 的状态。
- 复核 `v1.1-frontend-fidelity-implementation-spec.md` 是否与最终实现一致。
- 输出 frontend fidelity exit review。
- 独立门禁文档：`stage-gates/v1.1-e-exit-review.md`。

## 4. 验收标准

自动验收：

- 前端单元测试通过。
- Typecheck 通过。
- Build 通过。
- 页面内 E2E 通过。
- Playwright 截图基线生成并有对照记录。
- `scripts/capture_figma_make_baseline.mjs` 可运行，并生成 `current/capture-report.json`。

真实 Chrome 验收：

- 普通网页边缘出现浮动球。
- Hover 小长条出现并可打开面板。
- `440px` 展开态挤压网页。
- `50vw` 半屏态继续挤压网页。
- `>52vw` 覆盖态不继续挤压网页。
- 收起后网页布局恢复。
- Runtime online 后可读取页面、总结、问答。
- Runtime offline / missing context / tool failure 均可见。
- Mindmap / Mermaid artifact 视觉实现后续专项处理，不阻塞 V1.1-B/C。

视觉验收：

- `floating-default` 和 `floating-hover` 使用用户提供图片作为视觉参考。
- `panel-440-push`、`panel-50vw-push`、`panel-overlay`、`mobile-overlay` 以 PRD 硬约束作为验收依据。
- `runtime-offline` 单独设计验收。
- `artifact-mindmap` 后续专项处理。
- 当前截图与基线对照无关键布局偏差。
- 左轨、聊天主区、右工具区比例符合设计意图。
- 输入区固定，聊天区独立滚动。
- Artifact 区不破坏聊天布局。

## 6. 预审计意见

审计结论：

```text
Go for V1.1 documentation planning.
No implementation until Figma visual baseline and screenshot acceptance plan are available.
```

致命风险：无。

重大风险：

| 风险 | 等级 | 闭环要求 |
|---|---|---|
| Figma Make 只能读取资源清单，缺少截图 | 重大 | 进入实质前端开发前必须完成 Chrome CLI 截图硬切或普通 Figma `/design` 节点读取 |
| Chrome CLI 截到登录页/权限页/空白页 | 重大 | 不得纳入 `reviewed/`，必须由已登录 Chrome 手工补采或改用 `/design` 节点 |
| 视觉重构破坏 Runtime 链路 | 重大 | 每次样式/结构重构后复跑 PageContext、SSE、Mermaid、Session restore |
| 截图测试只覆盖 mock 页面 | 重大 | 必须同时覆盖真实 Chrome 普通网页 |
| Side Panel 被误判为高保真通过 | 重大 | 验收文档必须声明 Side Panel 只作调试入口 |
| 覆盖态底层网页交互未确认 | 重大 | 默认采用不可交互背景，除非 Figma 或人类确认 |
| 历史 `DEVELOPMENT_PLAN.md` 被误用 | 重大 | 该文件已标记为历史草案，V1.1 只按 PRD 与 stage-gate 执行 |

当前允许的下一步：

```text
只允许进入 V1.1 视觉规格冻结、Chrome CLI 截图硬切、截图人工复核和文档补齐。
在 Figma 视觉基线进入 reviewed/ 前，不允许进入 V1.1 实质前端实现，也不允许声明 V1.1 高保真通过。
```

## 7. Go / No-Go

Go：

- Figma 对照基线存在，且位于 `design/v1.1-figma-baseline/reviewed/` 或普通 Figma `/design` 节点可读取。
- PRD A-F 状态真实 Chrome 通过。
- Playwright 截图覆盖关键状态。
- Runtime / Trace / Session 链路不回退。
- Gap drawio 可打开且内容完整。

No-Go：

- 只能 Side Panel 通过。
- 没有视觉基线。
- 截图明显偏离关键布局。
- 页面内交互断裂。
- 网页布局无法恢复。
- Runtime / Trace / Session 断链。
