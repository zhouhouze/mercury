## 1. 阶段目标

V1.0-D 的目标是交付 V1 用户可安装的 Chrome 插件外壳，并打通当前网页到 Local Runtime 的 PageContext 链路：

```text
Chrome Load unpacked
-> Side Panel Chatbox shell
-> Runtime detection
-> Content Script 抽取当前页面
-> PageContext
-> POST /v1/page/context
-> Runtime activePage / event
```

---

## 2. PRD 规格检视

### 2.1 对应 PRD 目标

- 可在 Chrome 中安装的 Side Panel Chatbox。
- 当前网页上下文识别与提取。
- Chrome Side Panel 支持基础文字对话的前端承载。
- Runtime 不可用时插件明确提示。

### 2.2 必须保留的 V1 complete 目标

- Chrome 插件可通过 `Load unpacked` 安装。
- Side Panel 可打开。
- Side Panel 可展示 Runtime 连接状态。
- PageContext 可提交 Runtime。
- 后续 V1.0-E 可复用同一 Chatbox 消费 `/v1/chat/stream`。

### 2.3 本阶段非目标

V1.0-D 不做：

- 真实摘要 / 问答工具 UI 完整体验。
- Mindmap 渲染。
- YouTube / Monica 式站点内嵌。
- 网页贴边悬浮球。
- 浏览器自动操作。
- 长期记忆 / RAG。

## 4. 验收标准

- [ ] 插件可 build。
- [ ] 插件可通过 Chrome `Load unpacked` 安装。
- [ ] Side Panel 可打开。
- [ ] Runtime 未启动时 UI 明确提示。
- [ ] Runtime 启动后 UI 可重新连接。
- [ ] Content Script 可从真实网页抽取 title / url / domain / headings / cleanedText。
- [ ] PageContext 可提交 Runtime。
- [ ] Runtime 返回 `page_id` 和 `content_hash`。
- [ ] Runtime 记录 `page.context.received`。
- [ ] 不保存 AgentCore 核心状态到前端。
- [ ] 不引入网页自动点击或浏览器自动操作。
