# V1.0-E2 Stage Gate：Mermaid 视觉渲染补齐

版本：V1.0-E2 Mermaid Renderer Gate
日期：2026-06-02
阶段状态：计划与审计已闭环；实现完成；自动回归通过；真实 Chrome Mermaid 视觉复验待人工确认

---

## 1. 阶段目标

补齐 V1 mindmap 的前端可视化渲染，同时避免 Mermaid 依赖进入网页 content script 主包，防止 Chrome 扩展加载失败。

目标链路：

```text
Runtime 生成并验证 Mermaid source
-> artifact.created
-> 网页内面板插入 extension iframe renderer
-> iframe 渲染 Mermaid SVG
-> 失败时显示源码 fallback
```

---

## 2. PRD 规格检视

对齐项：

- V1 提供当前网页伴读和 Mermaid mindmap。
- Runtime 负责生成和校验 Mermaid source。
- 前端负责展示 Mermaid 可视化和源码 fallback。
- 未知 SSE event 不得导致 UI 崩溃。

非目标：

- 不引入新 event type。
- 不改变 ArtifactRecord 合同；Mermaid 继续使用 `metadata.format = "mermaid"`。
- 不引入 RAG、MCP、Skill、多 Agent、网络搜索或浏览器自动操作。

---

## 3. 开发计划

- 新增独立 extension page：`mermaid-renderer.html`。
- renderer page 内部加载 `mermaid` 并渲染 SVG。
- content script 仅创建 iframe，并通过 `postMessage` 传入 artifact source。
- renderer 通过 `postMessage` 返回成功/失败状态。
- content script 保留源码折叠区作为稳定 fallback。
- manifest 开放 renderer 页面和必要 chunks 给网页内 iframe 使用。

---

## 4. 验收标准

自动验收：

- `content-scripts/content.js` 严格 UTF-8 解码通过。
- `content-scripts/content.js` 不再回到 Mermaid 主包体积。
- Chrome extension tests 全绿。
- Typecheck 通过。
- Build 通过。

真实 Chrome 手工验收：

- 扩展可加载，无 manifest / UTF-8 错误。
- 点击 Mindmap 后，网页内面板可见 Mermaid SVG 或源码 fallback。
- Mermaid 渲染失败不影响继续对话。

防虚假验收：

- 仅生成 Mermaid source 不能声明视觉渲染通过。
- iframe 渲染失败时必须有可见源码 fallback。
- 不允许新增 ad-hoc event 字符串替代 `artifact.created`。

---

## 5. 审计意见

结论：

```text
Go for iframe renderer implementation.
```

致命问题：无。

重大风险：

| 风险 | 等级 | 闭环 |
|---|---|---|
| Mermaid 再次污染 content script 主包导致 Chrome 加载失败 | 重大 | renderer 独立 extension page，content script 只建 iframe |
| web accessible resource 配置缺失导致 iframe 无法加载 | 重大 | manifest 显式开放 renderer、assets、chunks |
| Mermaid 渲染失败被误判为通过 | 重大 | fallback 可见，视觉渲染和 source fallback 分开记录 |

闭环结论：

```text
无未闭环致命或重大规格偏差，可进入实质开发与验收。
```
