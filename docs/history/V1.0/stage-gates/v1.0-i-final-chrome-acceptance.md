# V1.0-I Stage Gate：最终 Chrome UI 验收

版本：V1.0-I Gate
日期：2026-06-02
阶段状态：历史 Side Panel 自动化验收脚本已完成；该口径已被 `PRD/窗口交互_PRD.md` 页面内交互验收覆盖

> 交互口径更新：2026-06-02 起，V1 前端页面体验以 `PRD/窗口交互_PRD.md` 为准。本文件记录的是历史 Chrome Side Panel 自动化边界，不能作为新口径 V1 complete 的最终验收依据。后续真实 Chrome UI 验收必须覆盖 PRD A-F 状态。

## 1. 阶段目标

本阶段历史目标是补齐旧口径 V1 complete 的最后高风险门槛：真实 Chrome / unpacked extension / Side Panel UI 验收。该目标已被新口径覆盖，后续最终 Chrome UI 验收必须覆盖 `PRD/窗口交互_PRD.md` A-F 状态。

必须验证：

- Chrome 加载 `apps/chrome-extension/chrome-mv3-unpacked`。
- Runtime 连接状态从 offline/checking 到 online。
- 真实网页 PageContext 可读取并提交。
- 历史 Side Panel Chatbox 可完成总结、页面问答、Mermaid mindmap。
- 历史刷新 Side Panel 后最近 session 可恢复。

---

## 2. PRD 规格检视

对齐 V1 PRD：

- Chrome Side Panel 不再是 V1 主前端形态；V1 主前端形态是 `PRD/窗口交互_PRD.md` 定义的网页内悬浮球与 AI 双轨面板。
- Runtime 是 AgentCore 状态源。
- 当前网页上下文进入 Runtime 后才能对话。
- Mermaid 必须真实渲染，或显示 source fallback。
- Session refresh 不丢是 V1 complete 硬门槛。

非目标：

- 不做网页悬浮球、站内嵌入式面板是历史口径；新口径必须做。
- 不做语音输入。
- 不做 RAG、长期记忆、MCP、Skill、多 Agent、浏览器自动操作。

## 3. 开发计划

### I.1 Chrome UI E2E 脚本

新增 `apps/chrome-extension/e2e/chrome-sidepanel-e2e.mjs`：

- 启动或复用 `127.0.0.1:17861` Runtime。
- 启动真实 Google Chrome persistent context。
- 加载 unpacked extension。
- 通过本地 HTTP server 提供真实 HTML fixture。
- 历史脚本尝试打开真实 Chrome Side Panel。
- 操作 UI 完成 PageContext、summary、question、mindmap、refresh recovery。

### I.2 防虚假验收规则

- 新口径不允许用 Side Panel、普通 extension page 或 debug page 替代网页内交互声明通过。
- 如果 `chrome.sidePanel.open()` 因用户手势或 Chrome 限制无法自动化，脚本必须返回 `MANUAL_REQUIRED`。
- 如果 Mermaid 只生成 source 但没有 render/fallback 可见，不得通过。
- 如果刷新恢复只来自前端本地状态，不得通过。

## 4. 验收标准

自动验收：

- `pnpm test` 通过。
- `pnpm run typecheck` 通过。
- `pnpm build` 通过。
- `pnpm run e2e:chrome` 通过，或明确返回 `MANUAL_REQUIRED`。

真实 UI 验收：

```text
Load unpacked
-> 打开真实网页
-> 历史 Side Panel online
-> 读取当前页面
-> 提交上下文
-> 总结
-> 页面问答
-> Mindmap 渲染或 fallback
-> 历史刷新 Side Panel
-> 最近 session 恢复
```

## 5. 预审计意见

审计结论：

```text
Go for validation automation.
```

致命问题：无。

重大风险：

| 风险 | 等级 | 闭环方式 |
|---|---|---|
| Chrome Side Panel 可能不暴露为 Playwright 可操作页面 | 重大 | 脚本以 `MANUAL_REQUIRED` 退出，不伪造通过 |
| `chrome.sidePanel.open()` 可能需要用户手势 | 重大 | 失败时进入人工验收，不降级为普通 extension page |
| 真实 Chrome UI 与 Playwright Chromium 行为差异 | 重大 | 使用 `channel: "chrome"` 启动本机 Google Chrome |

是否允许进入验收自动化：允许。

## 6. E2E 验收报告

已执行自动化回归：

```text

```

已新增并执行 Chrome UI E2E 脚本：

```text
apps/chrome-extension/e2e/chrome-sidepanel-e2e.mjs
pnpm run e2e:chrome
NAVIA_E2E_BROWSER=chromium pnpm run e2e:chrome
```

执行结果：

```text
Google Chrome:
MANUAL_REQUIRED: extension service worker was not exposed. pages=["about:blank"] workers=[]

Playwright Chromium:
MANUAL_REQUIRED: chrome.sidePanel.open failed: `sidePanel.open()` may only be called in response to a user gesture.

Playwright Chromium after _execute_action shortcut:
MANUAL_REQUIRED: keyboard-triggered extension action did not expose sidepanel.html.
```

结论：

- 自动化环境可以加载 Playwright Chromium 和扩展 service worker。
- Chrome Side Panel 本体没有作为 Playwright 可操作 page 暴露。
- `chrome.sidePanel.open()` 的用户手势限制是真实浏览器限制。
- 不能用普通 extension page 代替 Side Panel 通过验收。
- 本阶段自动化脚本达到可审计的高风险边界，必须进入人工真实 Chrome UI 验收。
