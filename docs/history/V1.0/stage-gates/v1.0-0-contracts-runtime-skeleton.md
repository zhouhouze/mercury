版本：V1.0-0 Gate Baseline

## 1. 阶段目标

V1.0-0 的目标是在写 AgentCore 业务 loop 之前冻结最小合同，形成后续 V1.0-A/B/C/D/E/G 可复用的 Runtime Skeleton。

本阶段不追求用户体验，不创建 Chrome 插件工程，不实现真实网页抽取，不接真实模型。它的价值是防止后续出现：

- 临时 AgentLoop。
- 临时 event dict。
- 工具自由文本返回。
- 没有 turn_id 的 chat。
- 只有实时流、没有 EventStore。
- API、状态机、数据模型、验收标准不一致。

---

## 2. PRD 规格检视

### 2.1 对应 PRD 目标

本阶段对应 `01-prd.md` 的 V1 目标：

- Contract-first Runtime Skeleton。
- AgentCore 使用可控、可观测的单 Session 状态机。
- Agent 状态机可视化、可验证、可观测、可扩展。
- Agent 具备预算、权限、上下文和本地文件访问监督机制。
- V1 不接 MCP / Skill / 长期记忆管理。

### 2.2 本阶段必须保留的 V1 complete 目标

虽然 V1.0-0 不实现插件，但合同必须支持最终 V1 complete：

```text
Chrome 插件安装
-> Side Panel 文字对话
-> /v1/chat/stream SSE
-> PageContext
-> AgentCore turn
-> ToolResult
-> Response / Artifact
-> Session Trace
```

因此 V1.0-0 必须提前定义：

- `session_id`。
- `turn_id`。
- `message_id`。
- `page_id`。
- `tool_call_id`。
- `artifact_id`。
- `event_id`。
- `trace_id`。
- `request_id`。

### 2.3 本阶段非目标

V1.0-0 不做：

- Chrome 插件 UI。
- Side Panel 交互。
- 真实模型调用。
- 真实网页 content script 抽取。
- Mindmap 模型。
- FunASR。
- SQLite 完整恢复。
- RAG / Memory Plane / MCP / Skill / Multi-Agent / browser automation。

### 2.4 规格偏差风险

| 风险 | 等级 | 处理 |
|---|---|---|
| 只写 Runtime demo，不冻结 schema | 致命 | 禁止进入 V1.0-A |
| API 返回自由结构，没有 envelope | 重大 | 必须补齐并测试 |
| SSE 事件不复用 AgentEvent envelope | 重大 | 必须统一 |
| EventStream 替代 EventStore | 重大 | 必须拆分接口 |
| ToolResult 未定义，工具返回自由文本 | 重大 | 必须补 ToolResult schema |
| 没有真实数据验收样本 | 重大 | 必须准备真实 HTML fixture 或真实 API event fixture |

## 4. 验收标准

### 4.1 基础合同验收

- [ ] API success envelope 可 schema validation。
- [ ] API error envelope 可 schema validation。
- [ ] ErrorCode enum 包含 V1 必需错误码。
- [ ] State enum 包含主路径和异常路径状态。
- [ ] Transition table 可生成 Mermaid。
- [ ] AgentEvent envelope 可 schema validation。
- [ ] ToolResult envelope 可 schema validation。
- [ ] Budget schema 可 schema validation。
- [ ] ID 规则覆盖 session / turn / message / page / tool / artifact / event / trace / request。

### 4.2 Runtime skeleton 验收

- [ ] Runtime 可启动。
- [ ] `/v1/health` 返回统一 envelope。
- [ ] `/v1/models/status` 返回 mock / rule_based / unavailable 等明确状态。
- [ ] `/v1/chat/stream` 返回 `text/event-stream`。
- [ ] SSE 事件复用 AgentEvent envelope。
- [ ] Runtime 默认绑定 `127.0.0.1`。
- [ ] CORS / Origin allowlist 有 skeleton 配置。

### 4.3 Store 与事件验收

- [ ] EventStore interface 存在。
- [ ] EventStream interface 存在。
- [ ] EventStore 可持久化或内存保存 event。
- [ ] EventStream 只负责实时推送。
- [ ] Trace 读取路径不得直接依赖 EventStream。

### 4.4 false-green 防线

以下任一出现即 V1.0-0 不通过：

- [ ] 没有 `turn_id`。
- [ ] 没有 ToolResult envelope。
- [ ] 没有 ErrorCode enum。
- [ ] 没有 EventStore。
- [ ] 只有 SSE，没有持久化事件接口。
- [ ] 工具或事件使用自由 dict，未受 schema 约束。
- [ ] 验收只使用 mock 或空数据。

本地校验命令：

```bash
python3 - <<'PY'
import json
from pathlib import Path
from jsonschema import Draft202012Validator, FormatChecker
base = Path('docs/navia_v1_project_docs/contracts')
checks = [
    ('api-response.schema.json', 'samples/api-success.json'),
    ('api-response.schema.json', 'samples/api-error.json'),
    ('page-context.schema.json', 'samples/page-context-article.json'),
    ('agent-event.schema.json', 'samples/agent-event-state-transition.json'),
    ('agent-event.schema.json', 'samples/eventstore-event.json'),
    ('tool-result.schema.json', 'samples/tool-result-summary.json'),
]
for schema_file, sample_file in checks:
    schema = json.loads((base / schema_file).read_text())
    sample = json.loads((base / sample_file).read_text())
    Draft202012Validator(schema, format_checker=FormatChecker()).validate(sample)
raw = (base / 'samples/chat-stream.sse').read_text().strip().split('\n\n')
for block in raw:
    lines = block.splitlines()
    event = lines[0].split(':', 1)[1].strip()
    data = json.loads(lines[1].split(':', 1)[1].strip())
    assert event == data['type']
    Draft202012Validator(json.loads((base / 'agent-event.schema.json').read_text()), format_checker=FormatChecker()).validate(data)
PY
```
