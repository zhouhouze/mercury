# V1.0-G Stage Gate：网页内 Session 恢复

版本：V1.0-G In-page Recovery Gate
日期：2026-06-02
阶段状态：计划与审计已闭环；自动回归通过；真实 Chrome 恢复复验待人工确认

---

## 1. 阶段目标

把历史 Side Panel 口径的 Session 恢复验收迁移到 `PRD/窗口交互_PRD.md` 的网页内面板口径。

目标链路：

```text
真实网页 -> 浮动球展开 -> 读取当前页面 -> 对话/Artifact
-> 刷新或重开网页 -> Runtime online -> 最近 session 恢复
-> activePage / messages / artifact 可见
```

---

## 2. PRD 规格检视

对齐项：

- 前端主体验必须是网页内浮动球与双轨 AI 面板。
- Chrome Side Panel 只保留为调试/兜底入口。
- 前端只缓存 `session_id`，AgentCore 状态必须来自 Runtime。
- 恢复内容必须来自 `GET /v1/sessions/{session_id}`，不能用前端本地假状态替代。

非目标：

- 不引入长期记忆、RAG、MCP、Skill、多 Agent 或浏览器自动操作。
- 不做跨设备云同步。

---

## 3. 开发计划

- 复用现有 `chrome.storage.local` 最近 `session_id` 缓存。
- 网页内面板 Runtime online 后调用 restore API。
- 恢复 activePage 摘要信息、历史消息和 artifact。
- 补充真实 Chrome 手工验收记录，作为当前自动化环境无法稳定驱动 extension service worker 的高风险流程证据。

---

## 4. 验收标准

自动验收：

- Runtime tests 全绿。
- Chrome extension tests 全绿。
- Typecheck 通过。
- Build 通过。

真实 Chrome 手工验收：

- 加载 `apps/chrome-extension/chrome-mv3-unpacked`。
- 打开真实网页。
- 读取当前页面。
- 发送总结或问答。
- 触发 mindmap artifact。
- 刷新页面或关闭后重开同网页。
- 展开 Navia 后恢复最近 session、页面信息、历史消息和 artifact。

防虚假验收：

- 不能用 Side Panel 替代网页内面板。
- 不能用前端内存里的未刷新状态替代 restore API。
- 缺少 Runtime online 时不得声明恢复通过。

---

## 5. 审计意见

结论：

```text
Go for implementation.
```

致命问题：无。

重大风险：

| 风险 | 等级 | 闭环 |
|---|---|---|
| 自动化环境不能稳定完成真实 Chrome content script 注入验收 | 重大 | 真实 Chrome 手工验收作为高风险流程，由人类执行 |
| 恢复数据可能来自前端临时状态 | 重大 | 验收必须刷新或重开页面后执行 |

闭环结论：

```text
无未闭环致命或重大规格偏差，可进入实质开发与验收。
```
