# V1.2-Closeout Draw.io Companion

配套图谱：

```text
docs/active/project/design/v1.2-ai-reading-automation-gap.drawio
```

对应分页：

```text
12 V1.2 Closeout 收关
```

## 1. 目标

本分页用于把 AC-Jumpback MVP 后的 P1 补强项固化为 V1.2 收关目标：

- 真实 Chrome Jumpback 截图级验收。
- A SourceRef / selector / textQuote / fallbackText 质量补强。
- C Mindmap 节点稳定绑定和同名节点消歧。
- B 来源证据面板、节点选中态和失败提示细化。
- 至少 20 页真实网页 / snapshot 收关矩阵。
- 最终 PRD 复检、false-green audit 和 V1.2 完成声明。

## 2. 颜色约定

| 颜色 | 含义 |
|---|---|
| 绿色 | 当前已完成或可复用能力 |
| 黄色 | 收关阶段需要修改或强化 |
| 蓝色 | 收关阶段新增的验收 / 证据 / 自动化 |
| 红色 | No-Go 或禁止声明范围 |

## 3. 目标架构

```text
真实网页
-> A SourceRef Quality
-> C Stable Mindmap Binding
-> D Artifact / Trace Boundary
-> B Side Panel Evidence UX
-> Content Script Jumpback
-> Screenshot Evidence
-> V1.2 Closeout Report
```

## 4. 当前架构差异

| 维度 | 当前 AC-Jumpback MVP | V1.2-Closeout |
|---|---|---|
| 真实截图 | 有 native Side Panel 主路径截图 | 必须证明点击后 DOM 高亮或 fallback |
| Jumpback 测试 | 组件 + content script DOM 测试 | 真实 Chrome 截图级结果 |
| SourceRef | 有 fallback / textQuote | 增加 selector/textQuote/fallback 指标 |
| Mindmap 交互 | 可点击来源卡片 | 节点选中态、同名节点消歧、证据面板细化 |
| 页面覆盖 | 5 页 native UX / 3 页 Jumpback MVP 目标 | 至少 20 页收关矩阵 |

## 5. 出门条件

只有以下全部满足才允许声明 V1.2 mock-first product path complete：

- 20 页收关矩阵通过。
- 5 个真实 Chrome Jumpback 截图级样本通过。
- 3 个 DOM highlight 样本和 2 个 fallback 样本可复核。
- 前端 / 后端 / A/C 模块测试、生产构建、HTML 报告器通过。
- PRD review 与 false-green audit 无 fatal / major。

## 6. 证据合同

收关阶段必须产出：

```text
report.json
acceptance-report.html
prd-review.md
false-green-audit.md
screenshots/
```

`report.json` 至少包含：

```text
schemaVersion
passed
summary.pagesTotal / pagesPassed
summary.chromeJumpbackSamples
summary.domHighlightedCount
summary.fallbackShownCount
quality.selectorAvailability
quality.textQuoteAvailability
quality.fallbackAvailability
quality.jumpbackCoverage
pages[]
jumpbackSamples[]
testCommands[]
claim
```

每个 Jumpback 样本必须有 before / after 截图、metadata、nodeId、sourceRefIds、attemptedStrategies、result 和 failureReason optional。

机器校验合同：

```text
docs/active/project/contracts/v1_2_closeout_report.schema.json
```

最终计入 `pagesTotal` 的页面必须有 `snapshotPath`，或 `evidenceMode=live_chrome` 且带完整 before/after 截图和 metadata。URL-only planned 页面不得计入。

固定验收命令：

```text
npm run e2e:chrome:jumpback-closeout
npm run e2e:chrome:jumpback-closeout:report
```
