# V1.4 Reading Map Development and Acceptance Plan

Date: 2026-06-24
Status: V1.4-0 documentation gate

---

## 1. Scope

V1.4 is a Side Panel reading map upgrade after V1.3 Evidence Card Mindmap. It keeps the current Chrome extension plus Python Runtime architecture and only changes the B renderer surface.

In scope:

- B-local ReadingMapViewModel derived from EvidenceCardViewModel.
- Two-column reading map view inside the existing mindmap artifact card.
- Node/theme selection, detail pane, source evidence, and jumpback/fallback status.
- Unit/component tests and real Chrome or reproducible snapshot acceptance evidence.

Out of scope:

- Full V1 completion.
- Final floating ball / hover strip as a completion claim.
- Canvas Knowledge Map.
- RAG, Memory, Web Research, PPT, Deep Research, multi-agent orchestration, voice, desktop pet, browser automation product features, or default local file access.

## 3. Acceptance Commands

Minimum automated checks:

```bash
npm --prefix apps/chrome-extension test -- mindmap_renderer ArtifactInlineCard
npm --prefix apps/chrome-extension run typecheck
python3 scripts/validate_v1_4_reading_map.py
```

Chrome acceptance should reuse the current extension build and real/snapshot page flow. If native Chrome Side Panel cannot be controlled reliably, the report must write a blocker or mark the sample as non-native; it must not pass by screenshot assumption.
