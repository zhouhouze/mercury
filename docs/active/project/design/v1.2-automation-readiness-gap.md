# V1.2 自动化开发就绪度 Gap

版本：V1.2 Automation Readiness Gap
日期：2026-06-04

## 1. 结论

当前 V1.2 文档已经具备模块拆分和架构方向，但进入“全部自动化开发”前必须完成合同可执行化、fixture 驱动验收、PRD 覆盖矩阵和 Integration 接线矩阵。

本轮补强后的目标状态：

```text
Go for staged automated implementation after V1.2-0 review.
```

不是一次性 Go for all modules。A/B/C/D 仍必须按 stage-gate 逐阶段或并行分支开发，并分别完成 mock 验收。

## 2. 当前 Gap

| Gap | 风险 | 补强文档 |
|---|---|---|
| Markdown 合同不可执行 | 字段偏移、可选性不一致 | module `executable-contract.md`、公共合同矩阵 |
| 模块入口未锁死 | 各 Codex 自行设计 API，联调返工 | module `public-api.md` |
| mock 验收不可执行 | false green | module `fixture-spec.md`、`test-and-evidence-plan.md` |

| 集成边界偏高层 | Integration Codex 重写模块逻辑 | `v1.2-integration-contract-matrix.md` |
| drawio 未覆盖 V1.2 | 新开发者无法通过图谱理解 A/B/C/D | `v1.2-ai-reading-automation-gap.drawio` |
| D 被误解为从零实现完整 AgenticLoop | 与 piAgent 选型冲突、Core 难替换 | CoreProvider ADR、D core provider docs |

## 3. P0 补强项

- 每个模块有 `public-api.md`。
- 每个模块有 `executable-contract.md`。
- 每个模块有 `fixture-spec.md`。
- 每个模块有 `test-and-evidence-plan.md`.
- 公共文档声明字段所有权。
- PRD coverage matrix 覆盖关键体验路径。
- Integration contract matrix 覆盖 A->D、D->C、D->B、B->Debug。
- V1.2 drawio gap 图谱可打开并包含 7 个页面。
- D 模块已明确 CoreProvider 可替换，piAgentProvider 为首选实现目标，MockCoreProvider 为测试 fallback。

## 4. V1.2 自动化开发 Go 条件

- V1.2-0 stage-gate 完成项目 owner review。
- A/B/C/D 模块内文档包完整。
- 所有模块 public API 与公共合同一致。
- 所有模块 fixture 和 evidence 路径明确。
- Integration Codex 只能做 wiring，不重写模块业务逻辑。
- drawio 图谱和 companion markdown 已登记。

## 5. No-Go 条件

- 任一模块缺少 public API。
- 任一模块缺少 fixture 或 mock 验收计划。
- PRD coverage matrix 缺少网页读取、总结、问答、mindmap、trace、offline/failure 任一主路径。
- A/C/D 直接改前端 renderer。
- B 直接调用 A/C/MCP/Skill/API。
- D 绕过 governance 执行 adapter。
- piAgent 或其他 CoreProvider 绕过 D Adapter Layer 直接写 Artifact、SSE、Trace 或 UI。
- Integration Codex 为了联调重写模块业务逻辑。

## 6. 审计意见

完成本补强包后，V1.2 文档可支撑：

- 模块级自动化实现。
- 模块级 mock 验证。
- 合同偏差审计。
- 少量 Integration wiring。

仍不允许：

- 不经 stage-gate 直接全量实现。
- 用 mock 声明真实 E2E 通过。
- 扩展到 V2 RAG、长期记忆、多 Agent 或浏览器自动操作。
