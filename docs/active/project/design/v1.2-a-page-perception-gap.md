## 1. 目标

本图谱只服务当前 `A-V1.2` 阶段，不再承载 A-V1.0 / A-V1.1 历史路线或 A-V1.12+ 未来媒体规划的独立页面。

A-V1.2 的阶段目标固定为：

```text
高质量网页感知
+ 结构化页面摘要
+ 可反跳证据
+ Debug 可验证 JSON
+ 100-page corpus gate
```

推荐组合路线固定为：

```text
DOM baseline
+ candidate extractor ensemble
+ A-owned schema normalization
+ SourceMap / jumpback
+ Quality Evaluator
+ DebugEvidenceBundle
+ 100-page corpus gate
```

## 3. 当前阶段边界

A-V1.2 只交付网页感知层：

- A 输入 `PageReadingInput` 或可复现 HTML snapshot。
- A 输出 `StructuredPageContext`、`HighSignalPageContext`、`PerceptionDigest`、`SourceMap / SourceRef`、`PagePerceptionQualityReport` 和 `DebugEvidenceBundle`。
- D/C/B 只能通过公共合同消费 A 输出，且 high-signal 输出只有在 `downstreamReadiness = pass` 时可作为主上下文。
- `degraded` 只能 fallback 或 Debug；`fail` 必须回退或返回 `PAGE_CONTEXT_REQUIRED`。

A-V1.2 不交付：

- 最终 assistant answer。
- Mindmap 生成。
- Artifact 创建。
- SSE、EventStore 或 Trace 写入。
- RAG、Notebook、Flashcards、Quiz、Podcast、长期记忆或多 Agent。
- OCR、VLM、ASR、视频或直播 engine 执行。

## 4. 验收口径

本图谱必须能让审计方快速回答：

- A-V1.2 当前能力与目标能力差异是什么？
- A-V1.2 的公共输出、质量门槛和 D/C/B 调用路径是什么？
- A-V1.2-0 到 A-V1.2-8 如何开发、验收和打回？
- 100-page corpus gate 如何防止 false green？
- 用户如何通过 Debug JSON、source fallback 和 quality state 判断 A 是否真的读懂网页？

## 5. No-Go

以下任一发生，A-V1.2 不得声明完成：

- 少于 100 个最终计入网页。
- URL-only、`planned` 或 `annotated` gold 页面计入最终通过率。
- `PerceptionDigestItem` 缺少 `sourceRefs`。
- `SourceRef` 缺少 `textQuote` 或 `fallbackText`。
- `quality-report.json` 写死 pass 或放宽公式过关。
- Debug JSON 不能解释 pass / degraded / fail。
- 第三方 extractor 原始输出进入 Navia 公共合同。
- 未批准依赖审计就安装或强依赖 extractor。
- A 生成回答、Mindmap、Artifact、RAG 或学习产物。
- A 执行 OCR、VLM、ASR、video 或 live engine。
