# V1.3 Evidence Card Mindmap Gap Companion

状态：Active V1.3 stage goal; V1.3-0 documentation and audit closure required
对应 drawio：`docs/active/project/design/v1.3-evidence-card-mindmap-gap.drawio`

---

## 1. 本阶段为什么存在

V1.2 已经完成 AI Reading mock-first product path 的基线：A 可产出高信号页面感知，C 可生成 digest-first Mindmap，D 可映射 Artifact / Trace，B 可完成 Side Panel 展示与基础 Jumpback。当前主体验仍是 Chrome 原生 Side Panel，不是最终网页内悬浮球 / 双轨面板。

当前主要缺口在体验层：

- Mermaid 默认图的视觉质感弱。
- 节点不像“可验证知识单元”，更像普通文本图。
- source evidence、quality、jumpback 状态没有成为导图的一等交互。
- 用户难以快速判断 AI 总结的依据和节点重要性。

V1.3 因此选择 `Evidence Card Mindmap`：让导图节点成为带来源、摘要、质量和交互状态的证据卡片。

本阶段已被确认为当前开发目标。已有 Evidence Card 基线和 C 语义标签压缩实现只能作为阶段输入；最终完成声明必须等 V1.3-0 到 V1.3-5 的文档、实现、真实网页验收和审计证据闭环。

## 2. 当前架构与目标架构差异

| 维度 | 当前 V1.2 | V1.3 目标 |
|---|---|---|
| 主渲染 | Mermaid visual / source fallback | Evidence Card Mindmap 主视图，Mermaid 降级 |
| 节点形态 | SVG / Mermaid 文本节点 | 卡片节点：标题、摘要、source count、confidence、标签 |
| 交互 | 点击 source card / jumpback | 点击节点即选中、展开证据、邻接高亮、定位状态反馈 |
| 来源证据 | source fallback panel | source evidence panel 成为节点交互的一等区域 |
| 视觉质量 | 默认图形质感 | 有层级、有密度、有方向感、有力反馈 |
| 长期演进 | 单页 artifact | 未来可演进为 Canvas Knowledge Map |

关键差异分层：

| 层级 | 当前架构 | 目标架构 | 需要验证 |
|---|---|---|---|
| 事实输入 | A digest / sourceRefs / qualityReport 可用 | 继续作为唯一事实来源 | A 不生成 Mindmap，C/B 不绕过 A/D |
| 生成层 | C 输出 Mermaid / nodeSourceMap | C 增强主题归并和语义标签压缩 | 长 digest 不进入卡片标题，source 保留 |
| 适配层 | D 映射 ToolResult / Artifact / Event / Trace | D 边界不变 | B/C 不直接写 Artifact 或 Trace |
| 渲染层 | Mermaid / Evidence Card 基线 | Evidence Card 为主视图 | B-local ViewModel、状态、降级可测 |
| 交互层 | source card / jumpback fallback | selected / neighbor / source panel / jumpback 状态闭环 | highlighted / fallback_shown / blocked 一致 |
| 验收层 | 单测、局部 E2E、恢复文档 | 8 页矩阵、3 个 native Side Panel 截图样本、HTML 报告 | JSON、截图、PRD review 一致 |

## 3. 目标调用关系

```text
A Page Reading
  HighSignalPageContext / PerceptionDigest / SourceMap / QualityReport
        |
        v
C Mindmap
  Mindmap tree / Mermaid source / nodeSourceMap / nodeBindings
        |
        v
D Adapter Layer
  ToolResult -> ArtifactRecord(type=mindmap) -> Event / Trace
        |
        v
B Mindmap Renderer
  EvidenceCardViewModel -> Evidence Card Mindmap -> source panel
        |
        v
Content Script
  user-triggered selector/domPath/textQuote jumpback
```

边界保持不变：

- A 不生成 Mindmap。
- C 不渲染 UI。
- D 是 Artifact/Event/Trace 唯一出口。
- B 不生成事实内容。
- Content Script 只执行用户触发定位。

## 5. 验收门槛和出门条件

必须满足：

- Evidence Card 是主视图。
- Mermaid 保留为 fallback / debug。
- 每个主要节点有 source 解释或明确 degraded reason。
- source evidence panel 可读。
- 节点点击、hover、selected、neighbor highlight 可见。
- 真实 Chrome Side Panel 截图证明用户路径。
- HTML 报告和 JSON 结论一致。
- 截图 metadata 能说明真实网页、Side Panel、节点、来源证据和 jumpback 状态。
- PRD review 明确 V1.3 只允许声明 Evidence Card Mindmap experience complete。

不得通过：

- 只改 Mermaid 主题。
- source fallback 冒充 DOM 成功。
- B 绕过 Runtime / D 生成 Mindmap。
- C 输出前端组件结构。
- V1.3 被扩大到 Canvas、RAG、Memory、Web Research 或 PPT。

证据包必须包含：

