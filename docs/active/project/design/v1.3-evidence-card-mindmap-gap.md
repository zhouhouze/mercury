# V1.3 Evidence Card Mindmap Gap Companion

状态：Conditional Go for V1.3-0 external audit
对应 drawio：`docs/active/project/design/v1.3-evidence-card-mindmap-gap.drawio`

---

## 1. 本阶段为什么存在

V1.2 已经完成 AI Reading mock-first product path：A 可产出高信号页面感知，C 可生成 digest-first Mindmap，D 可映射 Artifact / Trace，B 可完成 Side Panel 展示与基础 Jumpback。

当前主要缺口在体验层：

- Mermaid 默认图的视觉质感弱。
- 节点不像“可验证知识单元”，更像普通文本图。
- source evidence、quality、jumpback 状态没有成为导图的一等交互。
- 用户难以快速判断 AI 总结的依据和节点重要性。

V1.3 因此选择 `Evidence Card Mindmap`：让导图节点成为带来源、摘要、质量和交互状态的证据卡片。

---

## 2. 当前架构与目标架构差异

| 维度 | 当前 V1.2 | V1.3 目标 |
|---|---|---|
| 主渲染 | Mermaid visual / source fallback | Evidence Card Mindmap 主视图，Mermaid 降级 |
| 节点形态 | SVG / Mermaid 文本节点 | 卡片节点：标题、摘要、source count、confidence、标签 |
| 交互 | 点击 source card / jumpback | 点击节点即选中、展开证据、邻接高亮、定位状态反馈 |
| 来源证据 | source fallback panel | source evidence panel 成为节点交互的一等区域 |
| 视觉质量 | 默认图形质感 | 有层级、有密度、有方向感、有力反馈 |
| 长期演进 | 单页 artifact | 未来可演进为 Canvas Knowledge Map |

## 3. 目标调用关系

```text
A Page Reading
  HighSignalPageContext / PerceptionDigest / SourceMap / QualityReport
        |
        v
C Mindmap
  Mindmap tree / Mermaid source / nodeSourceMap / nodeBindings
        |
        v
D Adapter Layer
  ToolResult -> ArtifactRecord(type=mindmap) -> Event / Trace
        |
        v
B Mindmap Renderer
  EvidenceCardViewModel -> Evidence Card Mindmap -> source panel
        |
        v
Content Script
  user-triggered selector/domPath/textQuote jumpback
```

边界保持不变：

- A 不生成 Mindmap。
- C 不渲染 UI。
- D 是 Artifact/Event/Trace 唯一出口。
- B 不生成事实内容。
- Content Script 只执行用户触发定位。

## 5. 验收门槛和出门条件

必须满足：

- Evidence Card 是主视图。
- Mermaid 保留为 fallback / debug。
- 每个主要节点有 source 解释或明确 degraded reason。
- source evidence panel 可读。
- 节点点击、hover、selected、neighbor highlight 可见。
- 真实 Chrome Side Panel 截图证明用户路径。
- HTML 报告和 JSON 结论一致。

不得通过：

- 只改 Mermaid 主题。
- source fallback 冒充 DOM 成功。
- B 绕过 Runtime / D 生成 Mindmap。
- C 输出前端组件结构。
- V1.3 被扩大到 Canvas、RAG、Memory、Web Research 或 PPT。
