# V1.2-AC-Jumpback MVP Gap 图谱说明

对应 Draw.io：

```text
docs/active/project/design/v1.2-ac-jumpback-mvp-gap.drawio
```

## 1. 图谱目标

本图谱用于指导 `V1.2-AC-Jumpback MVP` 阶段开发。阶段目标是把 C 已经生成的 `MindmapNodeSourceMap`、`sourceRefIds`、`fallbackText` 和 `jumpback` 元数据转成用户可点击、可查看证据、可尝试定位的最小闭环。

它不用于声明：

- Monica 级复杂网页精准反跳完成。
- 完整 V1.2 complete。
- 完整 V1 complete。
- PDF / OCR / 视频 / 直播反跳完成。
- RAG、Memory、Web Research、PPT 或浏览器自动操作 ready。

## 2. 图谱页面

| 页 | 名称 | 用途 |
|---|---|---|
| 1 | 目标架构与当前差异 | 说明当前已有 source map，但缺点击反跳闭环 |
| 2 | 关键体验路径 | 说明用户从 Mindmap 节点点击到证据卡片 / DOM 高亮 / fallback 的路径 |
| 3 | 模块调用关系 | 说明 A/C/D/B/content script 的边界和调用方向 |
| 4 | 开发及验收计划 | 对齐 `V1.2-AC-Jumpback-0` 到 `V1.2-AC-Jumpback-5` |
| 5 | 验收门槛与出门条件 | 明确截图、真实网页、low-signal、false-green 和 No-Go |

## 3. 颜色含义

| 颜色 | 含义 |
|---|---|
| 绿色 | 当前已实现或可复用的能力 |
| 黄色 | 本阶段需要修改或增强 |
| 蓝色 | 本阶段需要新增 |
| 红色 | No-Go / 边界 |
| 灰色 | 后续阶段或不在本阶段范围 |

## 4. 关键设计判断

- C 已有反跳数据基础，不应把阶段目标写成“从零实现反跳”。
- 本阶段必须补齐的是产品级最小闭环：节点点击、证据卡片、DOM 尝试定位、失败 fallback。
- DOM 反跳成功不是唯一通过条件；fallback evidence 可读且失败原因明确也是合格降级。
- Content script 只响应用户触发，不做自动浏览器操作。
- 如果需要新增公共字段、事件类型或 artifact 类型，必须回到 `V1.2-0`。
