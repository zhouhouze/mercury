# V1.2-AC-Quality Draw.io Companion

状态：Active planning
日期：2026-06-14

## 1. 文档目的

本文件配套：

```text
docs/active/project/design/v1.2-ac-quality-hardening-gap.drawio
```

用于让项目成员快速理解 V1.2-AC-Quality 阶段的目标架构、当前差异、开发计划、验收门槛和出门条件。

## 2. 阶段边界

本阶段只聚焦：

- A 高质量网页感知。
- C digest-first Mindmap。
- AC 联动质量与 source fallback。
- 原生 Side Panel 中的 Debug / Mindmap 验收证据。

本阶段不做：

- 完整 V1.2 completion。
- A-V1.2 100-page production gate completion。
- 最终网页内悬浮球 / 双轨面板体验。
- RAG、长期记忆、多 Agent、浏览器自动操作、OCR/VLM/ASR/video/live engine。

## 3. Draw.io 页面说明

| 页名 | 说明 |
|---|---|
| `01 目标架构` | 展示 Chrome Side Panel、Runtime、A、D、C、B 的调用链路和模块边界 |
| `02 当前差异` | 对比当前实现和本阶段目标，标出 A/C/Debug/验收主要 gap |
| `03 开发与验收计划` | `V1.2-AC-Quality-0` 到 `-6` 的阶段顺序和出门条件 |

| `05 出门门槛` | 必须通过和 No-Go 条件，防止 false green |
| `06 项目里程碑` | 展示 V1.2-AC-Native、V1.2-AC-Quality、后续 V1.2-E Integration 和完整 V1/V1.2 声明边界 |

## 4. 关键审计问题

审计时必须确认：

- C 是否真正消费 A 的 `PerceptionDigest.items + SourceRef`。
- low-signal 页面是否 degraded / fail，而不是 pass。
- Debug 是否能解释 quality 和 source，不只是展示原始 JSON。
- Mindmap 主要节点是否有 `sourceRefIds` 或 `fallbackText`。
- 任何 summary / answer / Mermaid 是否都通过 Runtime / D Adapter，而不是 B 直接生成。

