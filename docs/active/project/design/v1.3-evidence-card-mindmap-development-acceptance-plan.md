# V1.3 Evidence Card Mindmap Detailed Development and Acceptance Plan

状态：Active execution plan after user approval of V1.3 drawio direction
日期：2026-06-23

---

## 1. 阶段目标

V1.3 的开发目标是让用户在 Chrome 原生 Side Panel 中获得可读、可交互、可追溯来源的 Evidence Card Mindmap 主体验。

本阶段只允许声明：

```text
V1.3 Evidence Card Mindmap experience complete.
```

本阶段不做：

- 完整 V1 网页内悬浮球 / 双轨面板。
- Canvas Knowledge Map。
- RAG、Memory、Web Research、PPT、Deep Research、多 Agent。
- 默认本地文件读取、浏览器自动操作、语音、桌宠。

---

## 2. 开发范围

### 2.1 Runtime / C Mindmap

允许：

- 改进 digest 节点语义标签压缩。
- 改进主题归并，避免长 digest 句子平铺。
- 保持 `nodeSourceMap`、`nodeBindings`、`sourceRefIds`、`textQuote`、`fallbackText` 可追踪。

禁止：

- C 输出 React / SVG / CSS 组件结构。
- C 读取 DOM 或调用 content script。
- C 创建 Artifact、Event、Trace。

### 2.2 Frontend / B Mindmap Renderer

允许：

- 从 `ArtifactRecord(type="mindmap")` 和 metadata 派生 B-local `EvidenceCardViewModel`。
- 渲染 Evidence Card Mindmap 主视图。
- 实现 hover、focus、selected、neighbor highlight、dimmed。
- 展示 source evidence panel。
- 复用 Mermaid visual/source 作为 fallback 或 debug。

禁止：

- B 直接调用 A/C/D 服务。
- B 生成事实内容、摘要、回答、Mindmap 或 Artifact。
- B 把 fallback 状态标记成 DOM success。

### 2.3 Content Script

允许：

- 仅在用户触发时执行 selector / domPath / textQuote 定位。
- 返回 `highlighted`、`fallback_shown` 或 `blocked`。

禁止：

- 自动浏览网页、自动点击网页、跨页面任务执行。

## 4. 最低测试矩阵

### 4.1 Unit / Contract

必须覆盖：

- `EvidenceCardViewModel` 可由 mindmap Artifact + `nodeSourceMap` 派生。
- `EvidenceCardViewModel` fixture 必须单独使用 `contracts/v1_3_evidence_card_mindmap.schema.json#/$defs/EvidenceCardViewModel` 验证；只验证最终 acceptance report 不足以通过 `V1.3-1`。
- 缺失 source 时节点显示 degraded reason。
- 重复 label 时节点可消歧，不错误映射来源。
- 长 label 被压缩，原始证据保留在 `fallbackText` 或 source item。
- Mermaid 渲染失败时显示 Mermaid source / source fallback。
- C 主题归并后仍保留 sourceRef 映射。

建议命令：

```bash
PYTHONPATH=services/local-runtime python3 -m pytest services/local-runtime/navia_runtime/modules/mindmap/tests
npm --prefix apps/chrome-extension test -- mindmap_renderer chat_renderer debug_renderer
npm --prefix apps/chrome-extension run typecheck
```

### 4.2 Build

必须通过：

```bash
npm --prefix apps/chrome-extension run build
```

WXT large chunk warning 只作为已知 warning，不单独阻塞 V1.3。

### 4.3 Native Side Panel / Visual Evidence

必须至少有：

- 8 个真实网页或可复现 snapshot 进入矩阵。
- 3 个真实 Chrome 原生 Side Panel 截图级样本。
- 1 个中文复杂页。
- 1 个技术文档页。
- 1 个 GitHub / README 页。
- 1 个长文或深层 heading 页。


已有证据包可以复用，但必须按当前 active 门槛重新验证。旧的 `passed=true` 不能自动等同于本轮 V1.3 完成。

### 5.1 Report Semantic Validation

JSON Schema 只能验证字段结构和最低数量门槛。V1.3 出门还必须执行语义校验：

- `passed=true` 时 `fatalIssues.length === 0`。
- `passed=true` 时 `majorIssues.length === 0`。
- `pagesPassed <= pagesTotal`，且 `pagesPassed` 满足当前阶段门槛。
- `nativeSidePanelSamples` 必须能回指 `screenshots[]` 中真实 native Side Panel 截图。
- 计入 native sample 的截图必须同时满足 `isNativeSidePanel=true`、`containsWebPageBody=true`、`containsNaviaPanel=true`、`containsEvidenceCardMindmap=true`。
- `visualEvidenceStatus="not_sampled"` 的页面只能作为 runtime / snapshot 矩阵项，不能计入 native Side Panel visual sample。
- fallback 样本不得被统计为 DOM highlight success。

语义校验失败时，即使 JSON Schema 通过，也不得声明 V1.3 complete。

如生成了 B-local ViewModel fixture，必须追加验证：

```bash
python3 scripts/validate_v1_3_evidence_card_mindmap.py --view-model-fixture <fixture.json>
```

## 6. PRD 体验闭环检查

V1.3 出门前必须逐项确认：

- 用户能读取当前网页。
- 用户能生成 Mindmap。
- Mindmap 主视图是 Evidence Card，不是 Mermaid 默认图。
- 用户点击节点后，节点 selected 状态明显。
- 邻接节点或边线高亮 / 非相关节点降权可见。
- source evidence panel 展示 `textQuote` 或 `fallbackText`。
- 用户触发定位时，DOM highlight success、fallback shown、blocked 可区分。
- Mermaid visual/source 只作为 fallback / debug。
- 低信号或缺 source 页面不会显示为正常成功。

---

## 7. 目标架构闭环检查

V1.3 出门前必须逐项确认：

- A 仍只输出 perception / digest / sourceRefs / qualityReport。
- C 仍只输出 mindmap tree、Mermaid source、nodeSourceMap、nodeBindings。
- D 仍是 ToolResult / Artifact / Event / Trace 唯一出口。
- B 只从 Artifact 和 metadata 派生 B-local ViewModel。
- Content script 只处理用户触发的定位和高亮。
- Runtime public contract 未因 V1.3 被扩大。
- `EvidenceCardViewModel` 未变成 A/C/D 的公共依赖。

---

## 8. 打回规则

- `V1.3-1`：无法从既有 Artifact 派生 ViewModel，或要求 C 输出前端组件结构。
- `V1.3-2`：卡片文本溢出、节点重叠、窄 Side Panel 不可读。
- `V1.3-3`：点击节点无法稳定打开 source panel，或 selected 状态不明显。
- `V1.3-4`：fallback 被标记为 DOM success，或失败原因不可见。
- `V1.3-5`：HTML 报告、截图、metadata、JSON 结论不一致。
