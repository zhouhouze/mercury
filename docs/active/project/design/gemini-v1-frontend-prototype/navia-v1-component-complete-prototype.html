伴随阅读产品的设计哲学

伴随式 Chrome 阅读助手应当对宿主页生态保持极高的敬畏。我们需要在保留原生页面完整性的同时，精确提供高追溯性的源证据视图。

精读时长 8 min 上下文：等待初始化 源节点追踪: 4

第一节：显式感知当前页面上下文

好的 AI 伴随助手永远不会让用户猜测“它是否已经读取了我的当前页”。我们需要显式传递已读就绪（Context Ready）信号，让阅读进度与对话上下文牢牢铆定。

当用户通过侧边栏对当前页提出细部观点质疑时，AI 提炼的核心证据环应该被精确关联（Grounded）到宿主 DOM 的段落中，绝不伪造高亮。

第二节：严格防范 False-Green 欺骗高亮

在极端的宿主网页安全或动态闭包阻断下，源证据追溯需要向用户表明真实的工程降维。

如果特定节点因 CSP 拦截导致无法在原文产生黄色高亮标记，前端渲染必须将该状态亮起醒目的琥珀色（Fallback）或红色（Blocked）警示，严禁模糊正则匹配滥竽充数。
