Designing reliable browser-side AI reading assistants

A companion reader needs to preserve the original page, keep evidence visible, and avoid turning focused reading into a noisy dashboard.

12 min read Current page context available Source refs: 18

Reading context

The assistant should make the page state obvious. Users need to know whether the current page has been read, whether a summary is grounded in page text, and whether a source can be located in the DOM or only shown as fallback evidence.

A side panel works best when the user can scan status, ask a question, inspect a structured artifact, and return to the article without losing orientation.

Implementation boundary

The frontend renderer should display artifacts returned by the local Runtime. It should not generate facts on its own or bypass Runtime contracts.

Evidence behavior

Source evidence has three user-visible outcomes: located highlight, fallback text, and blocked state. The interface must not label fallback as a successful DOM highlight.
