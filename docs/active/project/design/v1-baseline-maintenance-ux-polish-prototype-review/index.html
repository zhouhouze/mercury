Navia V1.0.x Baseline Maintenance + UX Polish 原型设计审查页

本页面用于给人类判断当前 Agent 是否完整理解前端页面优化诉求，并为后续自动化开发阶段提供实现方向和端到端验收参照。 它基于 active PRD、目标架构、BM/UX drawio 和 post-V1 frozen evidence 生成，不进入产品代码实现。

范围：launcher / sidebar / Chat / Mindmap / Source Evidence / Debug / Settings Frozen baseline：v1_post_v1_hardening 只读引用 No-Go：最终 Monica-like UX complete No-Go：RAG / Memory / Web Research / PPT / Deep Research

设计输入与图样生成说明

输入基线

本页以 active PRD 第 16 节、目标架构第 20 节、BM/UX stage gate、BM/UX drawio 八页结构和 post-V1 frozen evidence 为准。

图像生成状态

本机未发现 imag2 / image2 / gpt-image-2 终端命令。因此本页目标图样使用可复现 HTML/CSS 图形生成，不伪称 AI 文生图；真实截图只引用已有 evidence。

06 用户操作 / 体验路线图组

以下路径严格来自 PRD 和 BM/UX 目标，不增加也不减少承诺范围。

路径 1：启动与诊断

用户启动 Local Runtime；若在线，显示 Runtime 正常；若离线，显示 offline / reconnect 说明，不出现空白面板。

路径 2：默认贴边

用户打开普通网页，Navia 只显示贴边 launcher，不默认展开 sidebar，不强行遮挡正文。

路径 3：唤起面板

用户 hover/focus launcher 后看到完整入口；点击后展开右侧 sidebar，右侧 rail 可进入 Chat/Map/Src/Deb/Set。

路径 4：读取当前页

用户点击读取，页面上下文卡显示已解析状态；随后可总结、问答、生成 Mindmap。

路径 5：Mindmap 阅读

用户查看 Evidence Card Mindmap / Reading Map，选择节点后看到节点详情与证据关系。

路径 6：Source Jumpback

用户触发 source evidence：可定位时显示 marker；不可定位时显示 fallback；不可用时显示 blocked reason。

路径 7：布局恢复

用户 collapse / resize / drag 后，网页阅读区域可恢复，不发生永久错位或输入区不可用。

路径 8：Debug / Settings

用户需要诊断时进入 Debug/Settings，查看 Runtime endpoint、Debug JSON 和诊断信息；该路径保持 secondary。

08 对自动化实现与端到端验收的映射

子阶段 原型设计交付要求 端到端验收证据
BM-1 基线维护 冻结基线只读引用，新证据独立 build、post-V1 validator、Runtime health、敏感信息扫描
BM-2 启动诊断 Runtime offline / reconnect / Debug / report 入口清楚 Runtime online/offline 截图和 HTML 报告说明
BM-3 UX Polish launcher、sidebar、Chat、Mindmap、source evidence 目标设计落地 真实 Chrome 或 headless 截图证明关键路径可操作
BM-4 视觉回归 无明显遮挡、截断、虚影、焦点抢占、source 状态混淆 截图证据、screenshot metadata、false-green audit
BM-5 出门审计 HTML 报告可作为人类审计入口 PRD review、false-green audit、human spot-check，无 fatal / major
出门边界

本页只能支持“V1.0.x baseline maintenance and scoped UX polish”方向的实现与验收，不能支持最终 Monica-like UX complete、复杂站点全量高质量、V2 Memory/RAG 或 Web Research/PPT/Deep Research 声明。
