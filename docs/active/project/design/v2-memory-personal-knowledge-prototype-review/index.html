Personal Knowledge Base

本页用于审查 Navia V2 个人知识库目标体验：从当前网页保存到本地知识空间，形成可追踪、可删除、可授权的长期知识资产，并通过跨来源问答、Source Trace 和 Knowledge Graph 支持后续实现验收。

范围：V2 Memory / Personal Knowledge Base 形态：插件采集 + Knowledge Workspace 本地文件：仅显式授权 不代表 V2 已实现

01 / PRD 基线

V2 原型的事实边界

active PRD 当前只把 V2 登记为“本地备忘 / 个人知识库 / 标签化总结 / 类 RAG 蒸馏”，并要求重新评估 Runtime 主栈、Memory Plane、权限治理、数据保留和删除策略。因此本页先做原型审查，不声明开发完成。

Conditional Prototype

采用的 V2 范围

个人知识库：保存当前网页、管理来源、跨来源问答、source trace、知识图谱、删除/忘记、显式授权导入。

后端候选底座

data_service 可作为后端候选：workspace、source registry、session ingest/query、GraphRAG、LLMWiki、source trace、quality governance。

No-Go

不默认读取本地文件；不把 data_service console 当作 Navia 产品 UI；不声明 V2 ready；不引入 Web Research / PPT 生成 / Deep Research。

06 / 用户操作与体验路线图

不能超出也不能少于 V2 原型承诺

路线图覆盖 V2 个人知识库最小完整体验，不包含 Web Research、PPT 生成、Deep Research、媒体理解和自动浏览器任务。

路径 A：保存当前网页

1

用户在网页中打开 Navia launcher / sidebar。

2

点击“保存到知识库”，选择目标 workspace。

3

V2 Adapter 将 PageContext / SourceRef 映射为 data_service source。

4

Build 完成后显示 trace ready，可进入 Knowledge Workspace。

路径 B：跨来源问答与证据

1

用户进入 workspace ask 页面并选择 source 范围。

2

提交问题，返回 answer + evidence_refs。

3

点击引用打开 Trace Drawer，查看 source / unit / evidence。

4

无法精确定位时显示 fallback 或 blocked，不写成 located。

路径 C：本地文件显式授权

1

用户主动点击“授权本地目录”。

2

确认导入范围、用途、风险和撤销方式。

3

只扫描授权 roots，不读取未授权路径。

4

用户可撤销 root 并删除对应 source。

路径 D：删除 / 忘记

1

用户在 source detail 或 settings 中选择删除。

2

确认删除后，该 source 不再出现在 ask / graph / trace。

3

验收时必须再次查询，证明删除结果生效。

4

失败时显示 blocked，并给出下一步。

授权本地目录

Navia V2 默认不读取本地文件。只有在你显式授权后，指定目录才会作为 source root 进入知识库。你可以随时撤销授权并删除相关 source。

授权 root: ~/Documents/Research 能力: scan supported files only 撤销: supported 默认读取: disabled

删除 / 忘记 source

删除后，该 source 不应继续出现在跨来源问答、图谱、trace 或推荐列表中。后续验收必须用再次查询证明删除生效。
