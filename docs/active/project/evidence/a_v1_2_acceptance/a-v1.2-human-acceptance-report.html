验收结论

A-V1.2 已完成本阶段确定性基线开发。 当前通过范围是 dom_baseline + A-owned schema normalization + real snapshot corpus evidence。

这意味着 A 模块已经能在真实网页快照上稳定产出结构化页面、HighSignal、SourceMap、PerceptionDigest、QualityReport 与 DebugEvidence。

不包含：第三方 extractor ensemble、OCR/VLM/ASR、视频/直播理解、最终回答、Mindmap、ArtifactRecord、SSE、EventStore、Trace、RAG、MCP、Skill 或外部 API 调用。

验收门槛

✓
manifest_schema

Every page record validates as CorpusPageRecord.

✓
total_pages

Corpus has at least 100 page records.

✓
category_count

Corpus covers at least 10 categories.

✓
core_category_minimum

Each core category has at least 8 pages.

✓
snapshot_reproducibility

Every final counted page has a stored reproducible HTML snapshot.

✓
gold_review

Every final counted page has reviewed or semi-auto-accepted gold status.

✓
low_signal_expected_outcome

Low-signal pages must not be expected to pass.

✓
required_evidence_files

Every non-fail page has required evidence files, and fail pages have error evidence.

✓
quality_and_debug_evidence

Quality and Debug evidence satisfy source, grounding, jumpback, low-signal, and explanation gates.
