用户视角交互路径

  1. 打开一个真实网页快照标签页。
  2. 打开 Navia 扩展页面，确认 Runtime online，聊天页主操作可见。
  3. 点击「读取当前页面」，切换到 Debug 页确认页面状态变为 capture_ready。
  4. 点击「提交上下文」，确认 Runtime session 已创建，状态变为 captured。
  5. 点击「总结」，确认聊天区出现关键要点摘要。
  6. 输入「这个页面的核心目标是什么？」并发送，确认回答基于当前页面内容。
  7. 点击「Mindmap」，确认出现可视化思维导图 artifact。
  8. 刷新扩展页面，确认 Debug 页仍能恢复 captured 页面状态。

截图证据

1. 真实网页快照标签页

用于验收的页面不是空白 mock，而是通过本地 HTTP 服务打开的可复现 HTML snapshot。人类复核重点：页面标题与内容为浏览器扩展相关文章。

2. 聊天页初始态

扩展页面加载后，聊天页展示读取页面、提交上下文、总结、Mindmap 等操作。人类复核重点：主界面可用，非 Runtime offline。

3. Debug：读取页面后

点击「读取当前页面」后，Debug 页显示 capture_ready 和页面标题。人类复核重点：扩展确实读到了目标网页。

4. Debug：提交 Runtime 后

点击「提交上下文」后，Debug 页显示 captured 与已提交信息。人类复核重点：页面上下文已经进入 Runtime session。

5. 聊天：网页摘要

点击「总结」后，聊天页出现摘要和关键要点。人类复核重点：摘要内容来自当前页面，而不是空响应。

6. 聊天：基于页面内容问答

输入问题后，回答明确标注「基于当前页面」并引用页面主题。人类复核重点：问答链路可用，但本轮不评估大模型语义质量上限。

7. Mindmap artifact

点击「Mindmap」后，C 模块生成 Mermaid mindmap，B 侧渲染为可视化图谱。人类复核重点：截图为 artifact 区域直接截图，可以看到中心节点和多个分支节点。

8. 刷新恢复

刷新扩展页面后，Debug 页仍能看到 captured 的 activePage 状态。人类复核重点：不是纯内存 UI 假状态，刷新后仍可恢复。
