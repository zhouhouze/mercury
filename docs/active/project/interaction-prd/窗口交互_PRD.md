# AI 助手 PRD

## 0. 当前实现基线覆盖说明

截至当前 V1.x 前端基线，Chrome content script 页面内体验收敛为：**默认贴边 launcher、hover / focus 后弹出完整悬浮球、点击后展开右侧侧边栏聊天面板、再次点击后收起并恢复网页宽度**。

本文后续关于悬浮球默认态、hover 预展开态、点击悬浮球展开 / 收起、挤压 / 覆盖 / resize 的内容作为当前 V1 主线交互验收依据。当前阶段验收以“普通网页默认不被挤压，用户通过贴边 launcher 主动展开侧边栏，聊天、Debug、读取网页、总结、Mindmap 等入口均在侧边栏内完成”为准。

## 1. 产品定位

本产品第一版参考 Monica 的浏览器侧边栏交互，但只聚焦一个核心入口：**侧边栏唤起式 AI 聊天助手**。

第一版不做划词菜单、网页内右键菜单、全局搜索框、多窗口停靠等复杂入口。核心体验是：

- 插件开启后，页面边缘出现可移动悬浮球。
- 用户 hover 悬浮球后出现高亮边缘和伸出小长条。
- 用户点击小长条后，右侧展开 AI 双轨聊天界面。
- 窄距与半屏状态下，网页内容向左挤压。
- 超过半屏断点后，AI 面板切换为覆盖式显示。
- 用户点击悬浮球后，面板收起，仅保留悬浮球。

## 2. 核心界面状态

### 2.1 状态 A：悬浮球默认态

插件启用后，页面左右侧边缘出现 AI 悬浮球。悬浮球默认贴边，不展开，不遮挡主要内容。

```text
┌──────────────────────────────────────────────┐
│                                              │
│                 当前网页内容                  │
│                                              │
│                                              │
│                                              │
│                                      ◉       │
│                                  AI 悬浮球     │
│                                              │
└──────────────────────────────────────────────┘
```

交互规则：

- 悬浮球可上下拖动。
- 悬浮球支持贴左侧或右侧边缘。
- 拖动结束后自动吸附最近侧边缘。
- 悬浮球只负责移动、唤起和收起，不承担面板宽度调整。

---

### 2.2 状态 B：悬浮球 hover 预展开态

用户 hover 悬浮球时，悬浮球边缘高亮，并向页面内侧伸出小长条。

```text
┌──────────────────────────────────────────────┐
│                                              │
│                 当前网页内容                  │
│                                              │
│                                              │
│                              ┌────────────┐  │
│                              │  ◉   ⌘M    │  │
│                              └────────────┘  │
│                               hover 小长条    │
│                                              │
└──────────────────────────────────────────────┘
```

交互规则：

- hover 后悬浮球外圈高亮。
- 小长条从悬浮球向页面内侧伸出。
- 小长条作为主要点击热区。
- 小长条可展示快捷键提示，例如 `⌘M`。
- 鼠标离开后，小长条收回。

---

### 2.3 状态 C：窄距展开态，挤压网页

用户点击 hover 小长条后，右侧展开 AI 面板。窄距为面板最小宽度，网页内容向左挤压。

```text
┌──────────────────────────────┬──────┬────────────────────┬──────┐
│                              │      │                    │      │
│          当前网页内容          │ 左轨 │      聊天主区       │功能区 │
│        被向左挤压显示          │  ◉   │                    │      │
│                              │      │  ┌──────────────┐  │ 聊天 │
│                              │      │  │  对话消息区    │  │ 画图 │
│                              │      │  │  独立滚动      │  │ 视频 │
│                              │      │  │              │  │ 音频 │
│                              │      │  └──────────────┘  │ 发现 │
│                              │      │  工具栏 / 输入框    │ 更多 │
└──────────────────────────────┴──────┴────────────────────┴──────┘
```

布局规则：

- 窄距是 AI 面板展开后的最小宽度。
- 默认打开即进入窄距状态。
- 窄距状态采用挤压式布局，网页内容向左收缩。
- 面板不可继续向右拖拽缩小。
- 用户如需释放空间，只能点击悬浮球或收起按钮关闭面板。

建议参数：

| 参数 | 建议值 |
|---|---:|
| 窄距 / 最小宽度 | 440px |
| 左轨宽度 | 64-80px |
| 右侧功能区宽度 | 72-88px |
| 动画时长 | 200-260ms |

---

### 2.4 状态 D：半屏展开态，继续挤压网页

用户按住面板左边界向左拖拽，AI 面板变宽。拖到约屏幕一半时，进入半屏状态。

```text
┌──────────────────────┬──────┬────────────────────────────┬──────┐
│                      │      │                            │      │
│      当前网页内容      │ 左轨 │          聊天主区           │功能区 │
│      继续被挤压        │  ◉   │                            │      │
│                      │      │  更宽的对话、输入、工具空间   │      │
│                      │      │                            │      │
└──────────────────────┴──────┴────────────────────────────┴──────┘
```

交互规则：

- 面板左边界作为 resize handle。
- 用户向左拖拽时，AI 面板变宽，网页继续被压缩。
- 接近 `50vw` 时可以产生吸附。
- 半屏适合网页阅读、长文本问答、资料分析。

建议参数：

| 参数 | 建议值 |
|---|---:|
| 半屏断点 | 50vw |
| 吸附区间 | 48vw-52vw |
| 挤压最大宽度 | 50vw |

---

### 2.5 状态 E：宽工作区，覆盖网页

用户继续向左拉伸超过半屏阈值后，AI 面板不再继续挤压网页，而是切换为覆盖式显示。

```text
┌──────────────────────────────────────────────┐
│   当前网页内容保持底层布局                     │
│ ┌──────────────────────────────────────────┐ │
│ │ 左轨 │           AI 宽工作区          │功能区│ │
│ │  ◉   │                              │      │ │
│ │      │     长会话 / 写作 / 多工具      │      │ │
│ │      │                              │      │ │
│ └──────────────────────────────────────────┘ │
└──────────────────────────────────────────────┘
```

交互规则：

- 超过 `52vw` 后进入覆盖式。
- AI 面板继续向左扩展，但网页主体不再继续被压缩。
- 拖回 `48vw` 以下时，恢复挤压式。
- 覆盖态适合长会话、写作、复杂生成任务。

建议参数：

| 参数 | 建议值 |
|---|---:|
| 覆盖触发阈值 | >52vw |
| 覆盖回退阈值 | <48vw |
| 最大覆盖宽度 | 80vw |

---

### 2.6 状态 F：点击悬浮球收起

展开状态下，用户点击左轨中的悬浮球，AI 面板收起，网页恢复原始宽度，仅保留悬浮球。

```text
展开态
┌──────────────────────────────┬──────┬────────────────────┬──────┐
│          当前网页内容          │  ◉   │      聊天主区       │功能区 │
└──────────────────────────────┴──────┴────────────────────┴──────┘

点击悬浮球后

┌──────────────────────────────────────────────┐
│                                              │
│                 当前网页内容                  │
│                                      ◉       │
└──────────────────────────────────────────────┘
```

交互规则：

- 点击悬浮球收起整个面板。
- 收起后网页恢复原宽度。
- 保留悬浮球最近一次吸附边和垂直位置。
- 再次 hover / 点击小长条可重新展开。

## 3. 聊天窗口结构

### 3.1 双轨结构

展开后的 AI 助手由两层轨道组成：

```text
┌──────┬──────────────────────────────┬──────┐
│ 左轨 │          聊天工作区            │功能区 │
│      │                              │      │
│  ◉   │  聊天消息区                    │ 聊天 │
│      │  独立滚动条                    │ 画图 │
│      │                              │ 视频 │
│      │  功能栏                        │ 音频 │
│      │  输入框                        │ 发现 │
└──────┴──────────────────────────────┴──────┘
```

左轨：

- 展示 AI 悬浮球 / avatar。
- 作为面板收起入口。
- 保持固定宽度。

聊天工作区：

- 展示当前会话问答。
- 底部提供功能栏和输入框。
- 随面板宽度变化自适应。

功能区：

- 竖向功能入口。
- 第一版可先提供入口占位：聊天、画图、视频、音频、发现、备忘、更多。
- 功能区固定宽度，不参与面板拉伸。

---

### 3.2 聊天区域滚动条

聊天区域需要类似浏览器的纵向滚动条，用户可以拖拽查看当前会话历史。

```text
┌──────────────────────────────┬──┐
│ 用户问题                      │  │
│ AI 回复                       │  │
│                              │█ │  ← 聊天区独立滚动条
│ 历史消息                      │█ │
│                              │  │
└──────────────────────────────┴──┘
```

交互规则：

- 聊天消息区是独立滚动容器。
- 用户可通过鼠标滚轮、触控板、拖拽滚动条查看当前会话。
- 输入框、底部功能栏、右侧功能区固定，不随聊天内容滚动。
- 如果用户停留在底部，新回复自动滚动到最新。
- 如果用户正在查看历史，不强制跳到底部，显示“回到底部 / 最新回复”提示。
- 面板拉伸或切换覆盖态时，保持当前滚动位置。

## 4. 面板拉伸规则

### 4.1 拉伸入口

面板左边界为唯一宽度调整入口。

```text
网页内容 │ resize handle │ AI 面板
```

规则：

- hover 面板左边界时，鼠标变为横向 resize。
- 按住向左拖，AI 面板变宽。
- 向右拖最多回到窄距最小宽度。
- 双击边界恢复到窄距默认宽度。
- 系统记忆用户最近一次宽度。

### 4.2 三段式宽度策略

| 状态 | 宽度范围 | 网页处理 | 场景 |
|---|---:|---|---|
| 窄距 | 440px | 挤压网页 | 快速问答 |
| 半屏 | 约 50vw | 挤压网页 | 深度阅读、网页问答 |
| 宽工作区 | 52vw-80vw | 覆盖网页 | 写作、长任务、多工具 |

### 4.3 小屏兼容

当视口宽度不足时，挤压式体验可能导致网页不可用。

建议规则：

- 视口宽度 `< 900px`：禁用挤压式，直接采用覆盖式或全屏侧栏。
- 视口宽度 `< 720px`：AI 面板占满屏幕宽度，保留关闭入口。
