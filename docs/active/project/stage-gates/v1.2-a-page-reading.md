## 1. 阶段目标

把当前 PageContext 升级为高密度结构化网页事实源，为 D 的连续对话和 C 的 Mindmap 生成提供稳定输入。

A 模块是 AgentCore 的眼睛。A 负责识别网页和未来媒体环境中的可读事实，并把它们转成可追踪结构化上下文；A 不负责推理、最终回答、AgenticLoop、Artifact 创建、SSE 或外部工具执行。

A 模块专属图谱：

```text
docs/active/project/design/v1.2-a-page-perception-gap.drawio
docs/active/project/design/v1.2-a-page-perception-gap.md
```

必须输出：

```text
StructuredPageContext
ParagraphAnnotation[]
PageChunk[]
StructuredSummaryDraft optional
```

当前 A-V1.2 还必须以专属门禁为准：

```text
docs/active/project/stage-gates/v1.2-a-v1.2-production-page-perception.md
```

A-V1.2 目标输出包括：

```text
HighSignalPageContext
PerceptionDigest
SourceMap / SourceRef
PagePerceptionQualityReport
DebugEvidenceBundle
100-page corpus report
```

A 模块内部能力使用 `MODULE_VERSIONING.md` 定义的编号：

```text
A-V1.0-0：感知合同冻结
A-V1.0-1：文本 / DOM 结构识别
A-V1.0-2：图文网页识别
A-V1.0-3：OCR 识别规划
A-V1.0-4：表格 / 列表 / 代码块识别
A-V1.0-5：页面区域与信息密度识别
A-V1.1：High-Signal Page Perception 合同与 DOM baseline 能力
A-V1.2：Production Page Perception，高质量网页感知、结构化页面摘要、可反跳证据、Debug 可验证 JSON、100-page corpus gate
A-V1.12+：视频 / 直播等未来媒体感知规划
```

V1.2-A 完成不等于 V1.2 complete。A 只交付可被 Integration Codex 接入 D/C/B 的结构化感知能力和证据。
