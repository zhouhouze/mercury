# V1.2-AC Page Perception and Mindmap Bridge Stage Gate

版本：V1.2-AC Stage Gate
日期：2026-06-11
负责模块：A Page Perception、C Mindmap、D Adapter Layer、B Debug/Renderer Integration
状态：Automated contract, direct extension page E2E, and single-page visual screenshot acceptance passed; native Chrome Side Panel window automation and 3-page visual gate remain out of scope for this pass

---

## 1. 阶段目标

本阶段在当前 A-V1.2 与 C-V1.0 已有基线之上推进，不重开 V2，不扩大到 RAG、长期记忆、多 Agent 或浏览器自动操作。

目标链路：

```text
读取网页
-> A 生成 high-signal perception bundle
-> Runtime 主链路保存 / 返回可消费字段
-> Debug 页展示结构化感知、digest、source map、quality report
-> C 基于 A digest/source refs 生成 Mermaid mindmap
-> D 映射 ToolResult / Artifact / Event / Trace
-> B 展示 Mermaid 与 source fallback
```

完成后，用户和开发者应能在 Debug / 侧边栏内快速判断：

- A 是否真正提取出高质量结构化内容。
- C 的 mindmap 节点是否来自 A 的高信号事实。
- 每个主要节点是否能回到来源证据。
- 低信号或失败页面是否被正确降级，而不是生成假结果。

## 2. 当前架构差异

| 维度 | 当前实现 | 本阶段目标 |
|---|---|---|
| A 主链路 | `/v1/page/context` 主要保存 `StructuredPageContext` | 同步产出 high-signal perception bundle，并保持旧 activePage 兼容 |
| A 质量判断 | 107-page evidence 存在，但主链路可见性不足 | Runtime / Debug 可见 `qualityReport` 和 pass/degraded/fail 原因 |
| C 输入 | headingTree / paragraph fallback | `PerceptionDigest.items` 优先，headingTree fallback |
| C 来源 | paragraph/chunk source map | A SourceRef-backed nodeSourceMap，必须有 textQuote/fallbackText |
| D 职责 | 已有 Adapter Layer 和 `c_mindmap_adapter` | 继续作为唯一 ToolResult / Artifact / Event / Trace 映射出口 |
| B 职责 | 展示聊天、Debug、Artifact | 展示 A JSON、quality state、Mermaid 与 source fallback，不拥有 AgentCore 状态 |

## 4. 验收标准

### 4.1 A 主链路

- [ ] Runtime 主链路可返回或持久化 A perception bundle。
- [ ] 旧 `activePage` 字段保持兼容。
- [ ] `downstreamReadiness = pass/degraded/fail` 可被 D/C/B 读取。
- [ ] `fail` 不得触发假 summary、answer 或 mindmap。

### 4.2 C Mindmap

- [ ] C 优先消费 `PerceptionDigest.items`。
- [ ] headingTree / paragraph fallback 仅在 digest 不可用或 degraded 时使用。
- [ ] Mermaid validator 真实运行或 deterministic validator 有测试保护。
- [ ] repair 次数 `<= 1`。
- [ ] `metadata.nodeSourceMap` 覆盖 root 和主要节点。
- [ ] 每个主要节点有关联 SourceRef 或明确 fallback reason。

### 4.3 D / B Integration

- [ ] D 是唯一 ToolResult / Artifact / Event / Trace 映射出口。
- [ ] mindmap artifact 包含 `sourcePageId`、`turnId`、`toolCallId`。
- [ ] B 不直接调用 A/C 服务。
- [ ] Debug 页能展示 A JSON 和 C source map。
- [ ] 未知 SSE event 不导致 UI 崩溃。

### 4.4 真实数据

- [ ] 至少 12 个真实网页或可复现 snapshot。
- [ ] 至少 3 个真实 Chrome 页面。
- [ ] 至少 1 个中文复杂网页。
- [ ] 至少 1 个 low-signal 页面必须 degraded/fail。

## 5. 出门条件

可以进入下一阶段的条件：

```text
A 主链路可见
+ C digest-first
+ SourceRef-backed mindmap
+ Debug 可人工复核
+ Runtime trace 可审计
+ 真实网页验收通过
```

No-Go：

- A perception bundle 只存在离线 evidence，Runtime 主链路不可见。
- C 仍只基于 headingTree，却声明基于 A 高信号。
- nodeSourceMap 没有 sourceRef / textQuote / fallbackText。
- Debug 页不能解释 A 的质量判断。
- A/C 绕过 D 写 Artifact、SSE、EventStore 或 Trace。
- 使用 mock-only 数据声明通过。
- 没有真实网页或可复现 snapshot 证据，却声明 AC 端到端完成。
