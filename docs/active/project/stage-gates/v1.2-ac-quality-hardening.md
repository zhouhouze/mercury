# V1.2-AC-Quality Stage Gate: A/C 质量深化与真实网页扩展

状态：Planning ready for external audit
日期：2026-06-14

## 1. 阶段定位

V1.2-AC-Quality 承接 V1.2-AC-Native 的原生 Side Panel 稳定化成果，继续聚焦 A/C 两条主线：

- A：高质量网页感知、结构化页面摘取、可反跳证据、Debug 可验证 JSON。
- C：基于 A 的 `PerceptionDigest` / `SourceRef` / `QualityReport` 生成 digest-first Mermaid Mindmap。

本阶段不是完整 V1.2 出门，不替代 A-V1.2 100-page production gate，也不做最终网页内悬浮球 / 双轨面板体验。

## 2. 目标架构

```text
真实网页 / snapshot
  -> Chrome 原生 Side Panel
  -> B 触发读取 / Debug / Mindmap
  -> Runtime /v1/page/context
  -> A Page Reading Pipeline
      -> StructuredPage
      -> HighSignalPage
      -> PerceptionDigest
      -> SourceMap / SourceRef
      -> QualityReport
  -> D Adapter Layer
      -> ToolResult / Artifact / Event / Trace
  -> C Mindmap Generator
      -> digest-first Mermaid
      -> MindmapNodeSourceMap
      -> source fallback
  -> B Debug / Mermaid / Source Evidence
```

## 3. 当前差异

| 维度 | 当前事实 | 本阶段目标 |
|---|---|---|
| A 质量 | 已有 high-signal 和 low-signal degraded 基线 | 扩展真实网页矩阵，提升 digest 密度、噪声过滤和 quality 可解释性 |
| C 消费 | 已能生成 Mindmap，但需防止 heading-only 误声明 | readiness pass 时优先使用 digest/sourceRef；fallback 必须显式化 |
| Debug | 可展示基础状态和 JSON | 展示 quality、digest、sourceRef、nodeSourceMap、fallback reason |
| 反跳 | 已有 SourceRef / nodeSourceMap 基础 | 主要节点必须有 sourceRefId 或 fallbackText |
| 验收 | Native UX 5 页通过 | 至少 12 个样本矩阵 + 5 个原生 Side Panel 页面验收 |

## 4.1 样本矩阵

本阶段最小样本矩阵：

| 类别 | 最小数量 | 必须覆盖的风险 |
|---|---:|---|
| 中文复杂页 | 1 | 中文长文、复杂标题层级、导航噪声 |
| 图文混排页 | 2 | 图片 caption、alt、邻近文本、广告/推荐噪声 |
| 技术文档 | 2 | heading、代码块、列表、段落引用 |
| GitHub README | 1 | markdown 转 HTML、代码片段、目录噪声 |

## 5. 验收标准

必须通过：

- A Runtime 主链路返回 / 持久化 `structuredPage`、`highSignalPage`、`perceptionDigest`、`sourceMap`、`qualityReport`。
- C 在 `downstreamReadiness=pass` 时优先消费 `PerceptionDigest.items + SourceRef`。
- C fallback 时必须输出 `fallbackReason`。
- Mindmap 主要节点具备 `sourceRefIds` 或 `fallbackText`。
- Debug 展示 A quality、digest、sourceRef、C nodeSourceMap 和失败 / 降级原因。
- 至少 12 个真实网页或 snapshot 进入矩阵，至少 5 个真实 Chrome 原生 Side Panel 页面完成 E2E。
- HTML 报告和 false-green audit 可追溯。
- `report.json` 必须包含页面级和聚合级 `sourceCoverage`、`groundingCompleteness`、`jumpbackCoverage`、`lowSignalCorrectness`、`digestFirstUsage`。
- `acceptance-report.html` 必须包含每个样本的 URL、截图、quality 摘要、mindmap 结果、source fallback 和结论。

## 5.1 阶段打回规则

- 样本数量、类别或 low-signal / 中文复杂页缺失：打回 `V1.2-AC-Quality-1`。
- A quality 指标缺 numerator / denominator / method / threshold / passed，或 low-signal pass：打回 `V1.2-AC-Quality-2`。
- C heading-only 被声明 digest-first，或主要节点缺 sourceRef / fallbackReason：打回 `V1.2-AC-Quality-3`。
- Debug 仍依赖阅读原始大 JSON 才能判断质量：打回 `V1.2-AC-Quality-4`。
- 原生 Side Panel 证据缺截图、metadata、Runtime evidence 或结论不一致：打回 `V1.2-AC-Quality-5`。
- PRD 复检或 false-green audit 出现 fatal / major：打回对应子阶段。

## 6. No-Go

任一出现即不能声明本阶段完成：

- 只跑 A 离线 corpus，不跑 Runtime / Side Panel 主链路。
- 只有 URL 列表，没有可复核 snapshot、截图、Runtime evidence 或质量报告。
- C 仍只基于 headingTree，却声明 digest-first。
- Mindmap 节点没有 sourceRef / fallbackText / fallbackReason。
- Debug 仍只显示不可读原始 JSON。
- low-signal 页面被标记为 pass。
- A/C 绕过 D 写 Artifact、SSE、EventStore 或 Trace。
- B 前端直接生成 summary、answer 或 Mermaid。
- 引入 RAG、长期记忆、多 Agent、浏览器自动操作、OCR/VLM/ASR/video/live engine。
- 声明完整 V1.2 complete、完整 V1 complete 或 A-V1.2 100-page production gate complete。
