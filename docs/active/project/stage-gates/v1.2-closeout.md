# V1.2-Closeout Stage Gate: V1.2 收关与生产级完成声明

状态：Planning ready for external audit

## 1. 阶段定位

V1.2-Closeout 是 V1.2-AC-Jumpback MVP 之后的收关阶段。它不新增产品大能力，只把当前 A/C/D/B 主链路补强到可以谨慎声明：

```text
V1.2 AI Reading mock-first product path complete.
```

它不得用于声明完整 V1 complete，也不得引入 RAG、Memory、Web Research、浏览器自动操作、OCR/VLM/ASR、视频/直播理解、PPT、深度研究或多 Agent。

## 2. 阶段目标

必须完成：

1. 真实 Chrome 中的 Jumpback 截图级验收。
2. A SourceRef / selector / textQuote / fallbackText 质量提升。
3. 至少 20 个真实网页或可复现 snapshot 的收关样本。
4. Mindmap 节点点击、选中态、来源证据面板和失败提示细化。
5. 最终 PRD 复检、false-green audit 和 HTML 验收报告。

## 3. 目标架构

```text
真实网页
-> Chrome content script 抽取 PageContext
-> Runtime session.activePage
-> A high-signal bundle + SourceMap
-> C digest-first Mindmap + nodeBindings
-> D Artifact / Event / Trace 映射
-> B Side Panel 渲染 Mermaid / source cards
-> 用户点击节点或来源入口
-> content script selector/domPath/textQuote 定位
-> DOM highlighted 或 fallback_shown
-> HTML 报告记录截图和结论
```

## 6. 验收门槛

必须通过：

- 至少 20 个真实网页或可复现 snapshot。
- 至少 5 个真实 Chrome 原生 Side Panel 截图级 Jumpback 样本。
- 至少 3 个 DOM highlight 成功样本。
- 至少 2 个 fallback evidence 样本，且不得标记为 DOM success。
- 每个样本都有 URL、截图、metadata、nodeId、sourceRefIds、attemptedStrategies、result、failureReason optional。
- 前端测试、后端测试、A/C 模块测试、生产构建、HTML 报告器均通过。
- PRD 复检和 false-green audit 无 fatal / major。

样本计入口径：

- URL-only 页面只能计入当次真实 Chrome 现场验收样本。
- 需要长期回归、最终报告复验或声明 `pagesTotal` 的页面，必须有 `snapshotPath` 或等价保存的 HTML evidence。
- `pagesTotal >= 20` 的最终计数只能来自 `snapshotPath` 存在的页面，或在 `pages[].evidenceMode = "live_chrome"` 且有完整 before/after 截图和 metadata 的页面。
- planned / unreachable / screenshot-only / URL-only-without-Chrome-evidence 不得计入 `pagesPassed`。

## 7. No-Go

任一出现则不得声明 V1.2-Closeout 完成：

- 只用组件测试或 jsdom 测试声明真实 Jumpback 通过。
- 截图无法同时看到真实网页主体和右侧 Navia Side Panel。
- fallback evidence 被冒充为 DOM 高亮成功。
- 低信号页面被普通 pass。
- 同名节点跳到错误来源。
- A/C/B 绕过 D 写 Artifact / Event / Trace。
- 借本阶段引入 V2/V3/V4/V5 能力。
