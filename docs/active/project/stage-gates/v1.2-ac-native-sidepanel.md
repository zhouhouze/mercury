## 1. 阶段定位

V1.2-AC-Native 是 V1.2-AC 功能桥接之后的体验验收补强阶段。

本阶段不重做 A/C/D/B 功能，不扩大 V1.2 范围，只解决一个问题：

```text
已在 direct extension page 中跑通的读取、Debug、总结、问答、Mindmap，
必须能在真实 Chrome 原生 Side Panel 中被用户稳定使用和验收。
```

本阶段完成后只能声明：

- 用户可以在真实网页右侧打开 Navia 原生 Side Panel。
- Side Panel 内可以完成读取当前页、Debug 查看、页面总结、页面问答、Mindmap 与 source fallback。
- 所有通过声明都有截图、Runtime API 和测试证据支撑；未通过或无法自动化复核时必须输出结构化 blocker，不得伪 pass。

## 3. 目标架构

```text
Chrome Tab 当前网页
  -> Chrome extension action / keyboard command
  -> Chrome native Side Panel container
  -> B SidePanel Shell
  -> Background runtime proxy
  -> Runtime API Gateway
  -> D Adapter Layer
  -> A Page Perception / C Mindmap
  -> ToolResult / Artifact / Event / Trace
  -> B Chat / Debug / Mermaid / Source Fallback
```

边界：

- B 只渲染 Side Panel、Chat、Debug、Mermaid 和 source fallback，不拥有 AgentCore 状态。
- B 不直接调用 A/C/D 服务、MCP、Skill 或外部 API。
- A 只负责 page perception，不创建 Artifact、不发 SSE、不写 EventStore / Trace。
- C 只负责 mindmap 生成，不读取 DOM，不绕过 D。
- D 仍是 ToolResult / Artifact / Event / Trace 的唯一映射出口。

## 5. 验收标准

必须通过：

- Chrome / Chromium 安装 unpacked extension 后，普通网页可打开 Navia 原生 Side Panel。
- 通过截图必须同时包含真实网页主体内容和右侧 Navia Side Panel。
- Runtime online / offline 状态在 Side Panel 中可见。
- `读取当前页面`、`提交上下文`、`总结`、`Mindmap`、Debug 入口在 Side Panel 宽度内可见或可达。
- 读取当前页面后，Debug 能展示 activePage、A readiness、digest/source/quality 概览或明确失败原因。
- 同一原生 Side Panel 中至少完成一次页面总结和一次页面问答。
- 同一原生 Side Panel 中触发 Mindmap；成功时渲染 Mermaid，失败时展示 source fallback。
- 刷新或重开 Side Panel 后，session / activePage 可恢复，或展示明确恢复失败原因。
- 至少 3 个真实 Chrome 页面通过，其中至少 1 个中文复杂页面。
- 至少 1 个 low-signal 页面 degraded / fail，并在 Side Panel 中可见。

证据路径：

```text
docs/active/project/evidence/v1_2_ac/native-sidepanel-probe/
docs/active/project/evidence/v1_2_ac/native-sidepanel-ux/
```

证据输出必须分层：

```text
native-sidepanel-probe/report.json
native-sidepanel-probe/screenshots/*.png
native-sidepanel-probe/blockers/*.json

native-sidepanel-ux/report.json
native-sidepanel-ux/screenshots/*.png
native-sidepanel-ux/screenshots/*.metadata.json
native-sidepanel-ux/blockers/*.json
native-sidepanel-ux/false-green-audit.md
```

`native-sidepanel-probe/report.json` 只能说明原生 Side Panel 是否可打开。只有 `native-sidepanel-ux/report.json` 的 `passed=true`，或人工确认的等价截图证据，才能支撑本阶段体验通过声明。

每张计入通过证据的截图必须有同名 metadata JSON：

```json
{
  "screenshotPath": "screenshots/001-read-current-page.png",
  "pageUrl": "https://example.com/article",
  "tabTitle": "Example Article",
  "extensionId": "chrome-extension-id",
  "isNativeSidePanel": true,
  "containsWebPageBody": true,
  "containsNaviaPanel": true,
  "viewport": {"width": 1440, "height": 900},
  "panelApproxWidth": 420,
  "runtimeStatus": "online",
  "stage": "native-ux",
  "conclusion": "pass"
}
```

如果无法稳定打开、定位或复核原生 Side Panel，必须输出 structured blocker：

```json
{
  "blockerId": "native_blocker_001",
  "stage": "native-ux",
  "pageUrl": "https://example.com/article",
  "browser": "Chrome 版本号",
  "extensionId": "chrome-extension-id",
  "reason": "side_panel_not_opened",
  "evidencePaths": ["screenshots/001-before.png", "screenshots/002-after.png"],
  "attemptedActions": ["click_extension_action", "wait_for_side_panel"],
  "nextAction": "修复 background sidePanel.open 调用或改用可稳定触发的 command",
  "blocksCompletion": true
}
```

`reason` 只能使用以下枚举：

```text
side_panel_not_opened
cannot_distinguish_native_panel
button_unreachable
runtime_unavailable
screenshot_not_verifiable
chrome_automation_limitation
```

建议命令：

```text
pnpm --dir apps/chrome-extension build
pnpm --dir apps/chrome-extension e2e:chrome:native-probe
pnpm --dir apps/chrome-extension e2e:chrome:native-ux
```

`native-probe` 只能证明打开状态，不能证明完整用户体验。完整通过必须来自 `native-ux` 或人工确认的等价截图证据。

---

## 6. No-Go

任一出现即不能声明 V1.2-AC-Native 完成：

- 使用全屏 `chrome-extension://.../sidepanel.html` 截图声明原生 Side Panel 通过。
- 截图中没有真实网页主体和右侧 Navia Side Panel。
- 原生 Side Panel 无法稳定打开，且没有结构化 blocker。
- 关键按钮在 Side Panel 宽度下不可达。
- Debug 页不能解释 activePage、A quality、C source map 或失败原因。
- Mindmap 使用假数据或 heading-only fallback，却声明 digest-first native 验收通过。
- B 前端绕过 Runtime / D Adapter 直接生成总结、回答或 Mindmap。
- Runtime offline、PageContext missing、tool failure、Mermaid render failure 在 Side Panel 中不可见。
