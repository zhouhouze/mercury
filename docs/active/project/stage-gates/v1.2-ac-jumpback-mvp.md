# V1.2-AC-Jumpback MVP Stage Gate: Mindmap 来源反跳最小闭环

状态：Planning ready for external audit
日期：2026-06-16

## 1. 阶段定位

V1.2-AC-Jumpback MVP 承接 `V1.2-AC-Quality` 的 A/C 证据链成果，继续专注 C Mindmap 的用户可验证体验：用户点击 Mindmap 节点后，能看到该节点对应的来源证据，并在可行时让网页滚动 / 高亮到来源位置。

本阶段不是重新开发 A/C/D/B，也不是完整 Monica 级精准反跳系统。本阶段只做最小可用闭环：

```text
Mindmap 节点
-> nodeSourceMap
-> source evidence card
-> optional DOM scroll / highlight
-> fallback evidence when DOM jumpback fails
```

本阶段完成后只能声明：

```text
V1.2 Mindmap source fallback and basic jumpback MVP complete.
```

不得声明：

- Monica 级复杂网页精准反跳完成。
- 完整 V1.2 complete。
- 完整 V1 complete。
- PDF / 视频 / OCR / 直播反跳完成。
- RAG / Memory / Web Research / PPT / 浏览器自动操作 ready。

## 2. 目标架构

```text
真实网页
  -> A SourceMap / SourceRef
  -> C Mindmap Generator
      -> stable Mermaid node ids
      -> MindmapNodeSourceMap
      -> jumpback payload
  -> D Adapter Layer
      -> ToolResult / Artifact / Event / Trace 映射
  -> B Mindmap Renderer
      -> node click
      -> source evidence card
      -> content-script jumpback request
  -> Content Script
      -> selector / domPath / textQuote 定位
      -> scroll + highlight
      -> structured failure reason
```

## 5. 合同与边界

本阶段优先复用既有合同，不新增公共事件类型或 Artifact 类型。默认消费：

```text
ArtifactRecord.metadata.nodeSourceMap
MindmapNodeSourceMap[nodeId].sourceRefIds
MindmapNodeSourceMap[nodeId].paragraphIds
MindmapNodeSourceMap[nodeId].chunkIds
MindmapNodeSourceMap[nodeId].textQuote
MindmapNodeSourceMap[nodeId].fallbackText
MindmapNodeSourceMap[nodeId].jumpback
```

点击绑定、证据卡片和 content script 请求 / 响应必须遵守：

```text
docs/active/project/contracts/v1_2_adapter_contracts.md
13.1 V1.2-AC-Jumpback MVP 点击绑定合同
```

如果实现发现必须新增公共字段、事件类型或 API，则必须停止并回到 `V1.2-0` 合同冻结。

模块边界：

- C 只生成 Mermaid、stable node id、`nodeSourceMap` 和 jumpback payload。
- C 不读取 DOM、不调用 content script、不创建 Artifact、不发 SSE、不写 EventStore / Trace。
- B 只渲染 Mindmap、证据卡片和状态，不直接调用 A/C/D 服务。
- Content script 只执行用户触发的定位 / 高亮，不做浏览器自动操作编排。
- D 仍是 ToolResult / Artifact / Event / Trace 唯一映射出口。

## 6. 验收标准

必须通过：

- 每个可点击主要节点都能映射到 `nodeSourceMap[nodeId]`。
- 每个可点击主要节点必须具备 `MindmapNodeBinding`，且 `nodeSourceMapKey` 必须存在于 `nodeSourceMap`。
- 每个主要节点必须具备 `sourceRefIds` 或 `fallbackText`。
- 点击 Mindmap 节点后，Side Panel 必须显示来源证据卡片。
- 证据卡片必须展示节点标题、`textQuote` 或 `fallbackText`、sourceRefIds、fallback reason。
- `JumpbackRequest` 必须由用户点击触发，且必须记录 `requestId`、`nodeId`、`sourceRefIds`、`fallbackText` 和 `strategies`。
- 如果 `jumpback.mode="dom"`，content script 必须尝试 selector / domPath 定位。
- `JumpbackResult` 必须返回 `highlighted`、`fallback_shown` 或 `blocked`，并记录 attemptedStrategies。
- DOM 定位失败时，必须返回 structured failure reason，Side Panel 展示 fallback evidence，不得静默失败。
- 至少 3 个真实网页完成验收：中文复杂页、技术文档页、GitHub README 或长文页。
- 至少 1 个 low-signal / degraded 页面必须展示 fallback 或 unavailable 状态，不得伪 pass。
- HTML 验收报告必须包含截图、URL、点击节点、定位方式、成功 / 失败原因和最终结论。

## 7. No-Go

任一出现即不能声明本阶段完成：

- 只展示 Mermaid，不支持节点点击证据卡片。
- Mermaid 节点点击无法稳定映射到 `nodeSourceMap`。
- DOM 定位失败时不展示 fallback evidence。
- 使用坐标硬点或不可复现脚本冒充反跳通过。
- B 直接生成 Mindmap 或绕过 Runtime / D。
- C 直接读取 DOM 或调用 content script。
- Content script 执行非用户触发的浏览器自动操作。
- 借本阶段引入 RAG、Memory、Web Research、OCR/VLM、视频 / 直播理解、PPT 或深度研究。
- 声明 Monica 级复杂精准反跳、完整 V1.2 complete 或完整 V1 complete。
